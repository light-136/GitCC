## 1. 项目概述

| 属性 | 说明 |
|------|------|
| 项目名称 | SmartSemiCon |
| 项目全称 | Smart Semiconductor Controller |
| 项目定位 | 学习型半导体设备工业控制平台 |
| 目标用户 | 半导体设备软件工程师、自动化工程师、工业视觉工程师 |
| 开发语言 | C# 12 |
| 运行框架 | .NET 9 + WPF |
| 开发工具 | Visual Studio 2022 17.x+ |
| 操作系统 | Windows 10/11 64-bit |

**项目目标**：构建一套可独立运行的半导体设备工业控制平台原型。该平台涵盖运动控制、视觉检测、SECS/GEM 通讯、报警管理、配方管理等核心模块，全部采用模拟驱动实现，无需真实硬件即可完整运行和学习。

## 3. 五层架构说明

SmartSemiCon 采用经典的五层分层架构，依赖方向严格自上而下，所有层均可引用 Domain 层。

```
┌──────────────────────────────────────────────────┐
│                  UI 层 (WPF)                      │
│    MainWindow · ViewModels · Views · Themes       │
├──────────────────────────────────────────────────┤
│              Application 层 (应用逻辑)             │
│  StateMachine · Alarm · Recipe · User · Scheduler │
├──────────────────────────────────────────────────┤
│              Hardware 层 (硬件抽象)                │
│   Motion(轴管理/模拟卡) · Vision(相机/引擎)        │
│   VisionMotion(标定/飞拍/协同)                     │
├──────────────────────────────────────────────────┤
│            Infrastructure 层 (基础设施)            │
│  TCP通讯 · SECS/GEM · 事件总线 · 日志 · DI注册     │
├──────────────────────────────────────────────────┤
│               Domain 层 (领域核心)                 │
│     Enums · Models · Interfaces · Events          │
└──────────────────────────────────────────────────┘
```

### 3.1 各层职责

| 层 | 项目 | 职责 |
|----|------|------|
| **Domain** | SmartSemiCon.Domain | 定义枚举、模型（POCO）、接口契约、领域事件。纯净层，零外部依赖 |
| **Infrastructure** | SmartSemiCon.Infrastructure | TCP 通讯框架、SECS/GEM 协议实现、事件总线、日志服务、DI 注册扩展 |
| **Hardware** | SmartSemiCon.Hardware | 运动控制卡驱动（模拟）、轴管理器、相机管理器、视觉引擎、视觉-运动协同 |
| **Application** | SmartSemiCon.Application | 设备状态机、报警管理器、配方管理器、用户管理器、工业任务调度器 |
| **UI** | SmartSemiCon.UI | WPF 主窗口、10 个功能页面、暗色主题、MVVM ViewModel、值转换器 |

### 3.2 依赖关系图

```
UI ──→ Application ──→ Hardware ──→ Infrastructure ──→ Domain
 │         │              │              │
 └─────────┴──────────────┴──────────────┴──────→ Domain
```

- 所有层均引用 Domain 层（接口、枚举、模型）
- 上层可引用下层，下层不可引用上层
- 模块间通信通过事件总线（IEventBus）解耦

## 4. 核心功能模块列表

### 4.1 运动控制模块

| 功能 | 说明 |
|------|------|
| 多轴管理 | 支持 30+ 轴统一管理，按轴 ID 访问 |
| 运动模式 | 绝对定位、相对定位、JOG 点动、回原点 |
| 插补运动 | 直线插补（多轴同步）、圆弧插补 |
| 速度曲线 | 梯形速度曲线（加速→匀速→减速） |
| 安全保护 | 软限位保护、急停功能 |
| 模拟驱动 | SimulationMotionCard，无需硬件即可运行 |

### 4.2 视觉系统模块

| 功能 | 说明 |
|------|------|
| 相机管理 | 4 台相机（顶部/底部/侧面/定位），独立配置 |
| 视觉算法 | 模板匹配、Blob 分析、OCR、Mark 点定位、尺寸测量、缺陷检测 |
| 图像采集 | 软触发、硬触发、连续采集、编码器触发（飞拍） |
| 坐标标定 | 仿射变换、9 点标定法、像素→机械坐标转换 |

### 4.3 视觉-运动协同模块

| 功能 | 说明 |
|------|------|
| 视觉对位 | 拍照→检测→坐标转换→运动补偿 |
| 飞拍模式 | 边运动边拍照，编码器触发 |
| 九点标定 | 自动运动+拍照建立坐标映射 |

### 4.4 SECS/GEM 通讯模块

| 功能 | 说明 |
|------|------|
| HSMS-SS | 基于 TCP 的传输层协议实现 |
| SECS-II | 消息编解码，Stream/Function 消息体系 |
| GEM 状态机 | 通讯状态、控制状态（Online/Offline/Local/Remote） |
| 消息处理 | 注册式消息分发，自动回复机制 |

### 4.5 TCP 通讯框架

| 功能 | 说明 |
|------|------|
| 异步通讯 | 全异步 TCP 客户端/服务端 |
| 自动重连 | 断线自动重连，可配置间隔 |
| 心跳机制 | 定时心跳包检测连接活性 |
| 命令-响应 | 请求-回复匹配机制 |

### 4.6 应用层模块

| 模块 | 说明 |
|------|------|
| 设备状态机 | 完整的状态转换图，守卫条件，事件发布 |
| 报警管理 | 5 级报警（Info→Fatal），历史记录，一键复位 |
| 配方管理 | 工艺参数集合，点位表，配方切换 |
| 用户管理 | 3 级权限（操作员/工程师/管理员），SHA256 密码哈希 |
| 任务调度 | 4 级优先级线程池，周期性任务调度 |

### 4.7 UI 模块

| 页面 | 功能 |
|------|------|
| Dashboard | 设备总览仪表盘 |
| Motion | 运动控制操作面板 |
| Vision | 视觉系统操作面板 |
| SecsGem | SECS/GEM 通讯面板 |
| Communication | TCP 通讯监控 |
| Alarm | 报警管理面板 |
| Recipe | 配方管理面板 |
| Log | 日志查看面板 |
| Settings | 系统设置面板 |
| User | 用户管理面板 |

## 7. 学习价值

本项目涵盖半导体设备软件开发的核心知识体系：

1. **分层架构设计** — 五层架构、面向接口编程、依赖注入
2. **运动控制** — 多轴管理、梯形速度曲线、插补运动、软限位
3. **视觉系统** — 相机管理、算法管道、坐标标定、飞拍模式
4. **工业通讯** — TCP 通讯框架、SECS/GEM 协议栈
5. **设备管理** — 状态机设计、报警体系、配方管理、用户权限
6. **多线程** — 工业任务调度、生产者-消费者模式、线程安全
7. **WPF/MVVM** — 数据绑定、命令系统、暗色主题、导航框架
