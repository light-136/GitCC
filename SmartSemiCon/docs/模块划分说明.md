# SmartSemiCon 模块划分说明

> 文档版本：1.0  
> 更新日期：2026-05-09

## 1. 模块总览

```
SmartSemiCon
├── Domain 层 ——— 4 个子模块（枚举、模型、接口、事件）
├── Infrastructure 层 ——— 5 个子模块（TCP、SECS/GEM、事件总线、日志、DI）
├── Hardware 层 ——— 3 个子模块（运动控制、视觉系统、视觉运动协同）
├── Application 层 ——— 5 个子模块（状态机、报警、配方、用户、任务调度）
└── UI 层 ——— 4 个子模块（MainWindow、ViewModels、Views、Resources）
```

## 2. Domain 层模块

### 2.1 枚举模块 (Enums/DeviceEnums.cs)

定义系统所有状态和类型的枚举值，是整个系统的"语言基础"。

| 枚举名 | 值数量 | 用途 |
|--------|-------|------|
| `DeviceState` | 8 | 设备运行状态：Idle/Init/Auto/Manual/Paused/Alarm/EmergencyStop/Maintenance |
| `AxisState` | 8 | 轴运动状态：NotReady/Ready/Moving/Done/Error/Homing/Jogging/Disabled |
| `MotionMode` | 8 | 运动模式：Absolute/Relative/Jog/Home/线性插补/圆弧插补/电子齿轮/CAM |
| `AlarmLevel` | 5 | 报警等级：Info/Warning/Light/Heavy/Fatal |
| `ConnectionState` | 5 | 连接状态：Disconnected/Connecting/Connected/Reconnecting/Error |
| `GemControlState` | 5 | GEM 控制状态：三种离线 + 本地在线 + 远程在线 |
| `GemCommunicationState` | 3 | GEM 通讯状态：Disabled/WaitCommunicating/Communicating |
| `HsmsConnectionState` | 3 | HSMS 连接状态：NotConnected/Connected/Selected |
| `VisionEngineType` | 4 | 视觉引擎：Simulation/Halcon/OpenCV/VisionPro |
| `TriggerMode` | 4 | 触发模式：Software/Hardware/Continuous/Encoder |
| `UserRole` | 3 | 用户角色：Operator/Engineer/Administrator |
| `MotionCardType` | 4 | 控制卡类型：Simulation/EtherCAT/PLC/MotionBoard |

| 模型 | 属性数 | 说明 |

| `AxisStatus` | 11 | 轴状态：位置/速度/目标位置/状态/使能/限位/报警/时间戳 |
| `PointData` | 5 | 点位数据：名称/位置/速度/加速度/减速度 |
| `AlarmRecord` | 8 | 报警记录：代码/等级/消息/来源/触发时间/清除时间/操作者 |
| `LogEntry` | 5 | 日志条目：时间戳/级别/来源/消息/追踪ID |
| `UserInfo` | 7 | 用户信息：ID/用户名/显示名/角色/密码哈希/启用状态/最后登录 |
| `RecipeData` | 8 | 配方数据：ID/名称/版本/创建信息/参数字典/点位表/备注 |
| `CameraConfig` | 8 | 相机配置：ID/名称/连接串/分辨率/曝光/增益/触发模式/引擎类型 |
| `VisionResult` | 10 | 视觉结果：成功/像素XY/角度/分数/世界XY/耗时/错误/扩展数据 |
| `CalibrationData` | 7 | 标定数据：名称/相机ID/变换矩阵/像素比/时间/误差/标定点 |

### 2.3 接口模块 (Interfaces/CoreInterfaces.cs)

面向接口编程的核心，所有接口定义在 Domain 层，实现在各层中。

```
接口 → 实现关系：

IEventBus           → EventBusService         (Infrastructure)
ICommunicationService→ AsyncTcpClient          (Infrastructure)
ITcpServer           → AsyncTcpServer          (Infrastructure)
ISecsGemService      → SecsGemService          (Infrastructure)
IAxisController      → SimulationAxisController (Hardware)

ICamera              → SimulationCamera         (Hardware)

ILogService          → LogService               (Infrastructure)
```

### 2.4 事件模块 (Events/DomainEvents.cs)

所有事件继承自 `DomainEvent` 基类，携带 `EventId`、`Timestamp`、`Source` 三个公共字段。

**事件分类**：

| 分类 | 事件 | 触发时机 |
|------|------|---------|

| 运动控制 | `AxisStateChangedEvent` | 轴状态变更 |
| 运动控制 | `AxisAlarmEvent` | 轴发生报警 |
| 通讯 | `ConnectionStateChangedEvent` | TCP 连接状态变更 |
| 通讯 | `DataReceivedEvent` | 收到 TCP 数据 |
| SECS/GEM | `SecsMessageReceivedEvent` | 收到 SECS 消息 |
| SECS/GEM | `GemControlStateChangedEvent` | GEM 控制状态变更 |
| 报警 | `AlarmTriggeredEvent` | 报警触发 |
| 报警 | `AlarmClearedEvent` | 报警清除 |
| 视觉 | `ImageCapturedEvent` | 图像采集完成 |
| 视觉 | `VisionInspectionCompletedEvent` | 视觉检测完成 |
| 用户 | `UserLoggedInEvent` | 用户登录 |
| 配方 | `RecipeChangedEvent` | 配方切换 |

## 3. Infrastructure 层模块

### 3.1 TCP 通讯模块

#### AsyncTcpClient — 工业级异步 TCP 客户端

```
┌───────────────────────────────────────────────────────────┐
│                  AsyncTcpClient 架构                       │
│                                                           │
│  ┌─────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │ ConnectAsync │ →  │ TcpClient    │ →  │ NetworkStream│  │
│  │ (带超时)     │    │ (.NET Socket)│    │ (读写流)      │  │
│  └─────────────┘    └──────────────┘    └──────────────┘  │
│                                                           │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    三个后台循环                        │  │
│  │  1. ReceiveLoopAsync — 持续读取网络数据               │  │
│  │  2. HeartbeatLoopAsync — 定时发送心跳包               │  │
│  │  3. ReconnectLoop — 断线自动重连                      │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                           │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  命令-响应匹配                                        │  │
│  │  ConcurrentDictionary<int, TaskCompletionSource>     │  │
│  │  SendAndReceiveAsync → 注册TCS → 接收时TrySetResult  │  │
│  └──────────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────────┘
```

**关键配置参数**：

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `AutoReconnect` | true | 是否自动重连 |
| `ReconnectIntervalMs` | 3000 | 重连间隔（ms） |
| `HeartbeatIntervalMs` | 5000 | 心跳间隔（ms），0 禁用 |
| `ReceiveBufferSize` | 8192 | 接收缓冲区大小 |
| `NoDelay` | true | 禁用 Nagle 算法 |

### 3.2 SECS/GEM 模块

#### 协议栈分层

```
┌─────────────────────────────────────┐
│  GEM 层 — 设备行为规范               │
│  ┌─────────────┐ ┌───────────────┐  │
│  │GemControl   │ │GemCommunication│ │
│  │State(5状态) │ │State(3状态)    │  │
│  └─────────────┘ └───────────────┘  │
├─────────────────────────────────────┤
│  SECS-II 层 — 消息编解码             │
│  ┌─────────────┐ ┌───────────────┐  │
│  │SecsItem     │ │SecsFormat     │  │
│  │(数据项)     │ │(14种数据类型) │  │
│  └─────────────┘ └───────────────┘  │
├─────────────────────────────────────┤
│  HSMS 层 — TCP 传输                  │
│  ┌─────────────────────────────┐    │
│  │HsmsHeader (10字节固定头部)  │    │
│  │[SessionID 2B][Stream/Func]  │    │
│  │[PType/SType][SystemBytes 4B]│    │
│  └─────────────────────────────┘    │
│  消息格式：[4B长度][10B头][消息体]   │
└─────────────────────────────────────┘
```

**HSMS SType 控制消息类型**：

| SType | 含义 | 方向 |
|-------|------|------|
| 0 | Data Message（SECS-II 数据消息） | 双向 |
| 1 | Select.req（建立会话请求） | Equipment→Host |
| 2 | Select.rsp（建立会话响应） | Host→Equipment |
| 5 | Linktest.req（链路测试） | 双向 |
| 6 | Linktest.rsp（链路测试响应） | 双向 |
| 9 | Separate.req（断开请求） | 双向 |

**SECS-II 数据格式**：

| 格式代码 | 类型 | 用途 |
|----------|------|------|
| 0x00 | List | 容器，包含子项 |
| 0x20 | Binary | 二进制数据 |
| 0x24 | Boolean | 布尔值 |
| 0x40 | ASCII | ASCII 字符串 |
| 0x68 | Int2 | 2 字节整数 |
| 0x70 | Int4 | 4 字节整数 |
| 0x90 | Float4 | 4 字节浮点 |
| 0x80 | Float8 | 8 字节浮点 |
| 0xA4 | Uint1 | 1 字节无符号 |
| 0xB0 | Uint4 | 4 字节无符号 |

**超时参数**（符合 SEMI E37 标准）：

| 参数 | 默认值 | 说明 |
|------|-------|------|
| T3 | 45 秒 | 等待回复超时 |
| T5 | 10 秒 | 连接分离超时 |
| T6 | 5 秒 | 控制事务超时 |

#### SimulationVisionEngine — 模拟视觉引擎

6 种算法实现，返回带随机偏差的模拟结果：

| 算法 | 方法 | 模拟耗时 | 输出 |

### 4.3 视觉-运动协同模块

#### CoordinateTransformer — 坐标转换器

仿射变换矩阵：

```
[WorldX]   [a  b  c] [PixelX]
[WorldY] = [d  e  f] [PixelY]
[  1  ]   [0  0  1] [  1   ]
```

- `SetSimpleCalibration(pixelToMm)` — 简单缩放+偏移
- `CalibrateFromPoints(points)` — 多点标定（最小二乘法）
- `Transform(pixelX, pixelY)` — 像素→机械坐标转换

#### TriggerManager — 触发管理器

| 模式 | 方法 | 说明 |
|------|------|------|
| 软件触发 | `SoftwareTriggerAsync` | 停机拍照：到位→触发→等待 |
| 飞拍模式 | `FlyCaptureModeAsync` | 边运动边拍照，编码器位置触发 |

#### VisionMotionCoordinator — 协同控制器

**核心方法**：

1. `AlignAsync()` — 视觉对位
   ```
   采集图像 → Mark点检测 → 坐标转换 → 计算补偿 → 运动补偿
   ```

2. `NinePointCalibrationAsync()` — 九点标定
   ```
   For 3x3 网格:
     运动到标定位 → 拍照 → 检测Mark → 记录(像素, 机械)点对
   → 求解仿射变换矩阵 → 返回CalibrationData
   ```

## 5. Application 层模块

### 5.1 设备状态机 (DeviceStateMachine)

#### 状态转换图

```
                        ┌────────────────────────────────┐
                        │          EmergencyStop          │
                        │    (任何状态均可触发急停)         │
                        └────────────────┬───────────────┘
                                         │ Reset
                                         ▼
┌──────┐  Initialize  ┌──────┐  StartAuto  ┌──────┐
│ Idle │ ───────────→ │ Init │ ──────────→ │ Auto │
│      │ ←───────────│      │             │      │
└──┬───┘    Stop      └──┬───┘             └──┬───┘
   │                     │                    │  │
   │  StartManual       │  StartManual       │  │ Pause
   │                    │                    │  ▼
   │         ┌──────────┘              ┌────────┐
   │         ▼                         │ Paused │
   │     ┌──────┐                      │        │
   └────→│Manual│                      └───┬────┘
         │      │←─ Stop ─────────────────┘  Resume → Auto
         └──┬───┘
            │ Alarm
            ▼
        ┌──────┐  Reset (需报警已清除)  ┌──────┐
        │ Alarm│ ────────────────────→│ Idle │
        └──────┘                      └──────┘

        ┌──────────────┐  ExitMaintenance  ┌──────┐
        │ Maintenance  │ ────────────────→│ Idle │
        │              │ ←────────────────│      │
        └──────────────┘  EnterMaintenance └──────┘
```

#### 守卫条件

| 转换 | 守卫条件 |
|------|---------|
| Idle → Init | `ActiveAlarms.Count == 0` |
| Idle → Manual | `ActiveAlarms.Count == 0` |
| Alarm → Idle | `ActiveAlarms.Count == 0` |
| 其他 | 无额外守卫 |

### 5.2 报警管理 (AlarmManager)

#### 报警代码规范

```
1000~1999：运动控制报警
2000~2999：视觉系统报警
3000~3999：通讯报警
4000~4999：安全报警
5000~5999：流程报警
```

#### 报警等级与系统响应

| 等级 | 系统响应 | 日志级别 |
|------|---------|---------|
| Info | 仅记录日志 | Warning |
| Warning | 提醒操作员 | Warning |
| Light | 暂停流程，可自动恢复 | Warning |
| Heavy | 停止生产，需手动处理 | Error |
| Fatal | 立即急停，需设备维修 | Error |

#### 核心特性

- 同一报警代码不重复触发（`ConcurrentDictionary` 去重）
- 报警历史持久化（`ConcurrentBag` 存储）
- 触发/清除均发布领域事件
- 支持一键清除所有活跃报警

| `admin` | `admin123` | 管理员 | 完全权限，包括用户管理 |

#### 安全机制

- 密码使用 SHA256 哈希存储（`SHA256.HashData`）
- 权限检查：`HasPermission(UserRole)` — 当前用户角色 >= 要求角色
- 登录/登出发布领域事件

### 5.5 任务调度 (IndustrialTaskScheduler)

#### 优先级线程配置

| 优先级 | Thread.Priority | TaskCreationOptions | 典型周期 |
|--------|----------------|---------------------|---------|
| RealTime | Highest | LongRunning（专用线程） | 1~10ms |
| High | Normal | None（线程池） | 10~50ms |
| Normal | Normal | None（线程池） | 100ms+ |
| Background | Normal | None（线程池） | 1000ms+ |

#### 任务统计

每个 `ScheduledTask` 记录运行时统计：
- `ExecutionCount` — 累计执行次数
- `ErrorCount` — 累计错误次数
- `AverageExecutionTimeMs` — 指数移动平均执行耗时
- `LastExecuteTime` — 上次执行时间
