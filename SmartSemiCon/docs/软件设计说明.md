## 3. 核心接口定义

### 3.1 事件总线 IEventBus

```csharp
public interface IEventBus
{
    // 发布事件 — 通知所有订阅者
    void Publish<TEvent>(TEvent domainEvent) where TEvent : DomainEvent;

    // 同步订阅 — 返回IDisposable令牌用于取消订阅
    IDisposable Subscribe<TEvent>(Action<TEvent> handler) where TEvent : DomainEvent;

    // 异步订阅
    IDisposable Subscribe<TEvent>(Func<TEvent, Task> handler) where TEvent : DomainEvent;
}
```

### 3.2 通讯服务 ICommunicationService

```csharp
public interface ICommunicationService : IDisposable

    Task<byte[]?> SendAndReceiveAsync(byte[] data, TimeSpan timeout, CancellationToken ct = default);

    event EventHandler<byte[]>? DataReceived;
    event EventHandler<ConnectionState>? StateChanged;
```

## 4. 事件驱动架构说明

### 4.1 事件总线工作原理

```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│  运动控制    │ ──发布──→│              │←──订阅── │   UI 层     │
│  完成运动    │         │              │         │  更新界面    │
└─────────────┘         │   EventBus   │         └─────────────┘
                        │   Service    │
┌─────────────┐         │              │         ┌─────────────┐
│  报警管理    │ ──发布──→│              │←──订阅── │   日志服务   │
│  触发报警    │         │              │         │  写入日志    │
└─────────────┘         └──────────────┘         └─────────────┘
```

### 4.2 事件继承层次

```
DomainEvent (抽象基类)
├── EventId: Guid        ← 唯一事件ID
├── Timestamp: DateTime  ← 事件时间
└── Source: string       ← 来源模块

    ├── DeviceStateChangedEvent      ← 设备状态变更
    ├── AxisMotionCompletedEvent     ← 轴运动完成
    ├── AxisStateChangedEvent        ← 轴状态变更
    ├── AxisAlarmEvent               ← 轴报警
    ├── ConnectionStateChangedEvent  ← 连接状态变更
    ├── DataReceivedEvent            ← 数据接收
    ├── SecsMessageReceivedEvent     ← SECS消息接收
    ├── GemControlStateChangedEvent  ← GEM控制状态变更
    ├── AlarmTriggeredEvent          ← 报警触发
    ├── AlarmClearedEvent            ← 报警清除
    ├── ImageCapturedEvent           ← 图像采集完成
    ├── VisionInspectionCompletedEvent ← 视觉检测完成
    ├── UserLoggedInEvent            ← 用户登录
    └── RecipeChangedEvent           ← 配方切换
```

### 4.3 典型事件流示例

**场景**：运动完成 → 触发拍照 → 视觉检测 → 运动补偿

```
时间线 →

运动控制层:  MoveAbsolute(100mm) ──→ 到位 ──→ Publish(AxisMotionCompletedEvent)
                                              │
视觉系统层:                                    ├──→ Subscribe → CaptureAsync()
                                              │                    │
                                              │               Publish(ImageCapturedEvent)
                                              │                    │
视觉引擎层:                                    │                    ├──→ FindMarkAsync()
                                              │                    │         │
                                              │                    │    Publish(VisionInspectionCompletedEvent)
                                              │                    │         │
协同控制层:                                    │                    │         ├──→ Transform() → MoveRelative(补偿)
```

## 5. DI 容器配置说明

### 5.1 服务注册架构

```csharp
// App.xaml.cs 中的 DI 配置
var services = new ServiceCollection();

// 1. 日志框架 (Serilog)
services.AddLogging(builder => {
    builder.ClearProviders();
    builder.AddSerilog(dispose: true);
});

// 2. 基础设施层 (通过扩展方法统一注册)
services.AddInfrastructure();
//   → IEventBus       → EventBusService     (Singleton)
//   → ILogService     → LogService          (Singleton)
//   → ISecsGemService → SecsGemService      (Singleton)

// 3. 应用层
services.AddSingleton<IAlarmService, AlarmManager>();
services.AddSingleton<IUserService, UserManager>();
services.AddSingleton<IRecipeService>(sp => new RecipeManager(...));
services.AddSingleton<DeviceStateMachine>();
services.AddSingleton<IndustrialTaskScheduler>();

// 4. 硬件层
services.AddSingleton<AxisManager>();
services.AddSingleton<MotionScheduler>();
services.AddSingleton<CameraManager>();
services.AddSingleton<IVisionEngine, SimulationVisionEngine>();
services.AddSingleton<VisionMotionCoordinator>();

// 5. UI层 ViewModel (Transient — 每次请求新实例)
services.AddTransient<MainViewModel>();
services.AddTransient<MotionViewModel>();
// ... 其他 ViewModel
```

### 5.2 生命周期策略

| 注册方式 | 生命周期 | 适用场景 |
|----------|---------|---------|
| Singleton | 全局单例 | 事件总线、日志、报警管理、硬件管理器 |
| Transient | 每次新建 | ViewModel（每次导航创建新实例） |
| Scoped | 请求范围 | 本项目未使用（适用于 Web 场景） |

## 6. 线程模型说明

### 6.1 线程分类

```
┌─────────────────────────────────────────────────────────┐
│                    线程模型架构                           │
├─────────────────┬───────────────────────────────────────┤
│ UI 线程          │ WPF Dispatcher 线程，处理界面更新      │
│                 │ 所有 UI 操作必须通过 Dispatcher.Invoke │
├─────────────────┼───────────────────────────────────────┤
│ 实时线程         │ 运动控制状态刷新（10ms 周期）           │
│ (RealTime)      │ Thread.Priority = Highest             │
│                 │ TaskCreationOptions.LongRunning        │
├─────────────────┼───────────────────────────────────────┤
│ 高优先级线程     │ IO 监控、安全互锁（10~50ms）            │
│ (High)          │ 线程池线程                              │
├─────────────────┼───────────────────────────────────────┤
│ 普通线程         │ 视觉处理、通讯处理（100ms 级别）        │
│ (Normal)        │ 线程池线程                              │
├─────────────────┼───────────────────────────────────────┤
│ 后台线程         │ 日志写入、数据统计（秒级别）             │
│ (Background)    │ 线程池线程                              │
├─────────────────┼───────────────────────────────────────┤
│ TCP 接收线程     │ 每个 TCP 连接一个独立接收循环            │
│                 │ Task.Run + ReceiveLoopAsync            │
├─────────────────┼───────────────────────────────────────┤
│ 运动模拟线程     │ 每个运动指令一个独立线程                 │
│                 │ 10ms 更新周期模拟实时运动                │
└─────────────────┴───────────────────────────────────────┘
```

### 6.2 线程安全机制

| 机制 | 使用场景 |
|------|---------|
| `ConcurrentDictionary` | 报警管理、事务管理、消息处理器注册 |
| `ConcurrentBag` | 报警历史记录 |
| `lock` | 状态机状态转换、Socket 关闭操作 |
| `Interlocked` | 消息 ID 递增、SystemBytes 递增 |
| `CancellationToken` | 统一的线程生命周期管理 |
| `TaskCompletionSource` | 命令-响应匹配、异步等待 |
| `Dispatcher.Invoke` | UI 线程安全更新 |

### 6.3 线程交互示意

```
┌──────────┐   Publish()    ┌──────────┐  Dispatcher.Invoke  ┌──────────┐
│ 后台线程  │ ────────────→ │ EventBus │ ──────────────────→ │ UI 线程   │
│ (Motion)  │               │ (同步分发) │                    │ (刷新界面) │
└──────────┘               └──────────┘                    └──────────┘

┌──────────┐   WriteAsync   ┌──────────┐  DataReceived事件   ┌──────────┐
│ 发送线程  │ ────────────→ │ TCP流     │ ──────────────────→ │ 接收线程  │
│ (调用者)  │               │ (Network) │                    │ (循环读取) │
└──────────┘               └──────────┘                    └──────────┘
```

## 7. 关键设计模式

| 模式 | 应用位置 | 说明 |
|------|---------|------|
| **发布/订阅** | EventBusService | 模块间松耦合通信 |
| **策略模式** | IMotionCard / IVisionEngine | 不同硬件驱动可替换 |
| **工厂方法** | AxisManager.Configure | 根据 CardType 创建对应控制卡 |
| **观察者模式** | DataReceived / StateChanged 事件 | 状态变更通知 |
| **状态机模式** | DeviceStateMachine | 设备生命周期管理 |
| **命令模式** | RelayCommand (MVVM) | UI 按钮绑定到 ViewModel 命令 |
| **依赖注入** | Microsoft.Extensions.DI | 全局 IoC 容器 |
| **单例模式** | DI Singleton 注册 | 全局唯一服务实例 |
| **模板方法** | DomainEvent 基类 | 事件的公共字段和行为 |
| **门面模式** | AxisManager / CameraManager | 统一管理多个硬件实例 |
