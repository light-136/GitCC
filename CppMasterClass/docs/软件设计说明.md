# CppMasterClass 软件设计说明

## 1. 设计目标

构建一个可交互运行的 C++ 语言学习系统，覆盖 C++ 全部核心特性，每个特性模块独立可运行，输出直观的运行结果。

## 3. 设计原则

1. **零外部依赖** — 仅使用 C++ 标准库，降低编译门槛
2. **模块独立** — 每个 .cpp 文件导出一个 `void run_xxx()` 函数，互不依赖
3. **菜单驱动** — main.cpp 通过 `std::function` 统一调度，支持单个运行或全部运行
4. **渐进式学习** — 模块编号从基础到高级，符合学习路径
5. **实际输出** — 每个知识点都有 `std::cout` 输出，直观展示运行结果

## 4. 编译配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| C++ 标准 | C++20 | 最低要求 C++17 |
| 优化级别 | -O2 | 平衡性能与调试 |
| 警告级别 | -Wall -Wextra | 全量警告 |
| 线程库 | -pthread | 并发模块需要 |

## 5. C++20 降级策略

模块 04 中的 `concepts` 特性使用条件编译：
```cpp
#if __cplusplus >= 202002L
    // C++20 concepts 代码
#else
    // 降级提示
#endif
```

确保在 C++17 编译器上也能正常编译运行（仅跳过 concepts 演示）。
