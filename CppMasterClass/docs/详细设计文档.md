# CppMasterClass 详细设计文档

## 1. main.cpp — 菜单系统

### 数据结构
```cpp
struct MenuItem {
    int id;                      // 菜单编号
    std::string name;            // 显示名称
    std::function<void()> func;  // 模块入口函数
};
```

### 流程
```
启动 → 显示菜单 → getline(cin, input)
  → 去除\r → 检查EOF → 检查q退出
  → stoi解析 → id=0运行全部 / id=1~10运行指定
  → 循环回到菜单
```

### 设计决策
- 使用 `std::getline` 而非 `cin >>` 避免残留换行符
- 主动去除 `\r` 兼容 Windows 管道输入
- `stoi` 异常捕获处理非法输入

## 3. 02_pointers.cpp — 指针与引用

### 内存模型可视化
```
栈帧:
  x     = 42        [0x7fff1234]
  p     = 0x7fff1234 [0x7fff1238]
  ref   → x          (同一地址)

堆:
  new int(999)       [0x5678xxxx]  → delete 释放
  new int[5]         [0x5679xxxx]  → delete[] 释放
```

### 智能指针对比
```
unique_ptr<int> ──独占──▶ 堆对象
  └─ 不可拷贝，只能 move

shared_ptr<int> ──共享──▶ 堆对象 ◀── shared_ptr<int>
  └─ 引用计数 = 2          └─ 引用计数 = 2

weak_ptr<int> ──弱引用──▶ 堆对象
  └─ 不增加引用计数，lock()获取临时 shared_ptr
```

## 4. 03_oop.cpp — 面向对象

### 类层次结构
```
Vector2D
  ├─ 构造: 默认、参数化、拷贝、移动
  ├─ 析构: 输出日志
  ├─ 运算符: +, ==, <<(友元)
  └─ 方法: length(), getX(), getY()

Shape (抽象)
  ├─ 纯虚: area(), perimeter()
  ├─ 虚:   describe()
  ├──▶ Circle(radius)
  ├──▶ Rectangle(width, height)
  └──▶ Triangle(a, b, c)

Printable (接口)  ┐
                  ├──▶ DataPoint (多重继承)
Serializable (接口)┘
```

### 对象生命周期
```
进入作用域 → 构造(参数/默认)
赋值拷贝  → 拷贝构造
std::move → 移动构造(原对象被掏空)
离开作用域 → 析构(LIFO 逆序)
```

## 5. 04_templates.cpp — 模板

### 编译期 vs 运行时
```
模板实例化(编译期):
  maximum<int>(3,7)     → 编译器生成 int 版本
  maximum<double>()     → 编译器生成 double 版本
  Fibonacci<20>::value  → 编译器算出 6765

运行时(零开销):
  直接使用编译期生成的代码
  无虚函数、无类型检查开销
```

### SFINAE 工作原理
```
describe_type(42):
  尝试 is_integral<int>    → true  → 选中整数版本
  尝试 is_floating_point   → false → 替换失败(不是错误)

describe_type(3.14):
  尝试 is_integral<double> → false → 替换失败
  尝试 is_floating_point   → true  → 选中浮点版本
```

## 6. 07_concurrency.cpp — 并发

### 线程安全保证
```
ThreadSafeCounter:
  increment() {
    lock_guard<mutex> lock(mutex_);  ← 自动加锁
    ++value_;                         ← 安全修改
  }                                   ← 自动解锁(RAII)

atomic<int>:
  fetch_add(1, memory_order_relaxed); ← 硬件原子指令
  无锁，比 mutex 快
```

### 线程池架构
```
ThreadPool(4)
  ├─ worker 0 ──┐
  ├─ worker 1 ──┤
  ├─ worker 2 ──┼─── 竞争 tasks_ 队列 (mutex + cv)
  └─ worker 3 ──┘
                ↑
  submit(task) ─┘ 推入任务 + notify_one()
  ~ThreadPool()    → stop_ = true → notify_all() → join()
```

## 7. 10_real_world.cpp — 运动控制器

### 梯形速度规划
```
速度
 ^
 |    ┌──────────┐
 |   /            \
 |  /              \
 | /                \
 +───────────────────→ 时间
  加速   匀速   减速

算法:
  decel_dist = v² / (2a)
  if remaining <= decel_dist: 减速
  elif v < max_speed: 加速
  else: 匀速
```

### 搬运流程时序
```
时间→
axisX:  [---移动到100---] [-----移动到200-----] [---移动到0---]
axisY:  [---移动到50---]  [-----移动到100-----] [---移动到0---]
axisZ:             [↓取料] [↑]          [↓放料] [↑]
       ─────────────────────────────────────────────────────
       初始化    取料    搬运    放料    复位
```
