## 二、Domain 层详细设计

### 2.1 实体设计（Entities）

#### BaseEntity — 所有实体的抽象基类

```
BaseEntity（抽象类）
├── Id : Guid                    # 全局唯一标识（默认 Guid.NewGuid()）
├── CreatedAt : DateTime         # 创建时间（UTC）
├── UpdatedAt : DateTime?        # 最后更新时间
├── CreatedBy : string?          # 创建人
├── UpdatedBy : string?          # 更新人
├── IsDeleted : bool             # 软删除标记（查询自动过滤）
├── Version : byte[]             # 乐观并发令牌（EF Core RowVersion）
└── DomainEvents : List<DomainEvent>  # 待发布的领域事件暂存
```

**设计思路**：
- 所有实体继承此基类，统一审计和并发控制
- 软删除：数据库不物理删除，EF Core 全局查询过滤器自动排除 `IsDeleted=true`
- 领域事件暂存：实体行为方法中添加事件，SaveChanges 时统一发布

#### 实体清单

| 实体 | 核心字段 | 领域行为 |
|------|---------|---------|
| AxisConfig | AxisIndex, MaxVelocity, MaxAcceleration, SoftLimit± | ValidateParameters() |
| AlarmRecord | AlarmCode, Level, Category, Source, Message | Acknowledge(), Clear() |
| UserAccount | Username, PasswordHash, Salt, Role, Permissions | RecordLogin(), LockAccount() |
| RecipeData | Name, Parameters(Dict), VersionNumber, State | Activate(), Archive(), Clone() |
| CommunicationChannel | Protocol, Host, Port, ConnectionState | Connect(), Disconnect() |
| VisionTask | TaskType, CameraId, Threshold, Score | Execute(), SetROI() |
| CollaborationSession | OwnerUserId, ConnectedUsers, SharedState | Join(), Leave(), UpdateState() |

### 2.2 值对象设计（Value Objects）

**Position3D**（三维坐标）

```
Position3D (record struct)
├── X : double
├── Y : double
├── Z : double
├── operator + (a, b) → new Position3D(a.X+b.X, ...)
├── operator - (a, b) → 同上
├── operator * (p, scalar) → 缩放
├── DistanceTo(other) → Math.Sqrt(dx²+dy²+dz²)
├── Lerp(other, t) → 线性插值
└── ToString() → "(X, Y, Z)"
```

**MotionProfile**（运动参数）

```
MotionProfile (record)
├── Velocity : double           # 目标速度 (mm/s)
├── Acceleration : double       # 加速度 (mm/s²)
├── Deceleration : double       # 减速度 (mm/s²)
├── Jerk : double               # 加加速度 (mm/s³)，0表示梯形
└── SettlingTime : double       # 到位稳定时间 (ms)
```

**VisionRegion**（ROI区域）

```
VisionRegion (record)
├── X, Y : double               # 左上角
├── Width, Height : double      # 尺寸
├── Angle : double              # 旋转角度(°)
├── Contains(point) → bool
├── Intersects(other) → bool
└── Expand(margin) → new VisionRegion
```

### 2.4 接口设计（Interfaces）

```csharp
{
    // 发布事件（所有订阅者都会收到）
    Task PublishAsync<T>(T domainEvent) where T : DomainEvent;
    
    // 订阅事件（异步处理器）
    void Subscribe<T>(Func<T, Task> handler) where T : DomainEvent;
    
    // 订阅事件（同步处理器）
    void Subscribe<T>(Action<T> handler) where T : DomainEvent;
    
    // 取消订阅

}
```

### 2.5 规约模式（Specifications）

```
Specification<T>（抽象基类）
├── ToExpression() → Expression<Func<T, bool>>    # 表达式树（EF Core可翻译为SQL）
├── IsSatisfiedBy(entity) → bool                  # 内存判断
├── And(other) → AndSpecification<T>              # 与组合
├── Or(other) → OrSpecification<T>                # 或组合
└── Not() → NotSpecification<T>                   # 非

具体规约示例：
├── ActiveAlarmSpecification         → alarm.IsActive == true
├── AlarmByCategorySpecification     → alarm.Category == target
├── UserByRoleSpecification          → user.Role == target
├── ActiveRecipeSpecification        → recipe.State == Active
└── AlarmByDateRangeSpecification    → alarm.OccurredAt between start and end
```

### 3.3 事件总线实现

```
EventBusService : IEventBus
├── _handlers : ConcurrentDictionary<Type, List<WeakReference<Delegate>>>
│
├── Subscribe<T>(handler)
│   ├─ 获取或创建 handlers[typeof(T)]
│   ├─ 添加 WeakReference(handler)      # 弱引用防止内存泄漏
│   └─ lock 保护列表操作
│
├── PublishAsync<T>(event)
│   ├─ 获取 handlers[typeof(T)]
│   ├─ 遍历所有订阅者
│   │   ├─ 检查 WeakReference 是否存活
│   │   ├─ 存活 → 调用 handler(event)
│   │   └─ 死亡 → 标记清理
│   ├─ 异常隔离：单个handler异常不影响其他
│   └─ 清理已死亡的弱引用
│
└── Unsubscribe<T>(handler)
    └─ 从列表移除匹配的引用
```

```
              |       |          矩阵             |       |
   P7(u7,v7) ● ─ ─ ─ ● P8      求解      W7     ● ─ ─ ─ ● W8
              |       |                           |       |
   P9         ● ─ ─ ─ ●                   W9     ● ─ ─ ─ ●

   变换公式：
   ┌ x ┐   ┌ a11  a12  a13 ┐   ┌ u ┐
   │ y │ = │ a21  a22  a23 │ × │ v │
   └ 1 ┘   └  0    0    1  ┘   └ 1 ┘

   最小二乘法求解（≥3组点，9点过采样提高精度）
   RMS误差评估标定质量
```

#### 图像处理算法（纯数学实现）

```
ImageProcessor（不依赖第三方视觉库）
├── 灰度直方图    → 256个bin统计
├── 二值化
│   ├── 固定阈值  → pixel > threshold ? 255 : 0
│   └── 大津法    → 自动寻找类间方差最大的阈值
├── 均值滤波      → 3×3/5×5邻域均值
├── 高斯滤波      → 加权邻域（σ计算权重矩阵）
└── 边缘检测
    └── Sobel算子  → Gx/Gy 卷积核 → Magnitude = √(Gx²+Gy²)
```
