## 1. Application 层详细设计

### 1.1 设备状态机（DeviceStateMachine）

**文件位置**：`src/SmartIndustry.Application/StateMachine/DeviceStateMachine.cs`

**设计目标**：管理设备全局运行状态的有限状态机（FSM）。

**状态转换表**：

| 当前状态 | 允许转换到 |
|---------|-----------|
| Idle | Initializing |
| Initializing | Ready, Faulted |
| Ready | Running, Maintenance, Idle |
| Running | Paused, Stopping, Faulted, EmergencyStop |
| Paused | Running, Stopping, EmergencyStop |
| Stopping | Ready, Faulted |
| Faulted | Idle, EmergencyStop |
| Maintenance | Ready, Idle |
| EmergencyStop | Idle |

**线程安全**：使用 `SemaphoreSlim` 保证同一时刻只有一个状态转换在执行。

**回调机制**：支持 `OnEnter` / `OnExit` 回调注册，用于在状态变更时执行硬件初始化/清理动作。

### 1.2 报警服务（AlarmService）

**文件位置**：`src/SmartIndustry.Application/Alarm/AlarmService.cs`

**设计目标**：实现 `IAlarmService` 接口，提供报警全生命周期管理。

**双重存储策略**：
- **内存缓存**（`ConcurrentDictionary`）：快速读取活跃报警，UI 实时刷新
- **数据库持久化**（`IAlarmRepository`）：历史查询、统计分析

**报警定义预注册**：系统启动时调用 `RegisterAlarmDefinition()` 注册所有已知报警码，运行时 `TriggerAlarm()` 只需传入报警码即可获取预配置的等级和类别。

**防重复触发**：同一报警码的活跃报警不会重复创建。

### 1.3 配方服务（RecipeService）

**文件位置**：`src/SmartIndustry.Application/Recipe/RecipeService.cs`

**设计目标**：实现 `IRecipeService` 接口，管理工艺配方的 CRUD 和版本控制。

**版本管理流程**：
```
CreateRecipe("产品A") → v1 (Draft)
Activate(v1) → v1 (Active)
CreateNewVersion(v1) → v2 (Draft), v1 仍然 Active
Activate(v2) → v2 (Active), v1 自动 Archive
```

**激活排他性**：同一时刻同名配方只能有一个 Active 版本，激活新版本时自动归档旧版本。

### 1.4 用户服务（UserService）

**文件位置**：`src/SmartIndustry.Application/User/UserService.cs`

**权限模型（RBAC）**：

| 角色 | 权限范围 |
|------|---------|
| Viewer | 仪表盘/报警/日志/配方查看 |
| Operator | + 报警确认、配方加载、点动、回零、视觉采集 |
| Engineer | + 配方编辑/创建、运动配置、视觉标定、通信配置 |
| Administrator | + 用户管理、配方删除、系统设置 |

**Token 管理**：使用 GUID Token + 内存字典，8 小时过期，支持定期清理。

### 1.5 任务调度器（AppTaskScheduler）

**文件位置**：`src/SmartIndustry.Application/TaskScheduler/AppTaskScheduler.cs`

基于 `System.Threading.Timer` 的轻量级调度器，支持：
- 周期性重复任务（如 Token 过期清理、设备状态巡检）
- 一次性延迟任务
- 任务执行统计（执行次数、错误次数、上次执行时间）

### 1.6 SECS/GEM 服务（SecsGemService）

**文件位置**：`src/SmartIndustry.Application/SecsGem/SecsGemService.cs`

遵循 SEMI E30 标准的三层状态模型：
- **Communication State**：Disabled / Enabled / WaitCRA
- **Control State**：EquipmentOffline / OnlineLocal / OnlineRemote
- **Processing State**：Idle / Setup / Ready / Executing / Paused

支持设备变量（SV）注册和更新、集合事件（CEID）触发、SECS 消息处理器注册。

### 1.7 自动化服务（AutomationService）

**文件位置**：`src/SmartIndustry.Application/Automation/AutomationService.cs`

流程步骤编排引擎，支持：
- 有序步骤执行（每步有独立超时）
- 多循环运行（maxCycles 参数）
- 暂停/恢复/停止控制
- 步骤失败策略（StopOnFailure）
- 执行结果统计（完成步数、耗时、失败步骤定位）

## 2. DI 容器注册全景

```csharp
// Infrastructure 层
services.AddInfrastructure("Data Source=smartindustry.db");
  → AppDbContext (Scoped)
  → GenericRepository<T> / AlarmRepository / UserRepository (Scoped)
  → EventBusService / IEventBus (Singleton)
  → SerilogService / ILogService (Singleton)
  → FileService / IFileService (Singleton)
  → JsonConfigService (Singleton)
  → HttpClientFactory + HttpClientService (Singleton)

// Hardware 层
services.AddHardware();
  → SimulationMotionCard / IMotionCard (Singleton)
  → TrapezoidalProfile / SCurveProfile (Singleton)
  → LinearInterpolator / CircularInterpolator (Singleton)
  → AxisManager (Singleton)
  → MotionScheduler (Singleton)
  → CalibrationService (Singleton)
  → SimulationVisionEngine / IVisionEngine (Singleton)
  → SimulatedIoDevice / DigitalIoManager (Singleton)
  → VisionMotionCoordinator (Singleton)

// Application 层
services.AddApplication();
  → DeviceStateMachine (Singleton)
  → AlarmService / IAlarmService (Singleton)
  → RecipeService / IRecipeService (Scoped)
  → UserService / IUserService (Singleton)
  → AppTaskScheduler (Singleton)
  → SecsGemService (Singleton)
  → AutomationService (Singleton)
```
