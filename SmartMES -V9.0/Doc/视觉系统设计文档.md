# SmartMES V9.0 视觉系统设计文档

## 1. 概述

SmartMES V9.0 的视觉系统是一个完全自包含的工业视觉解决方案，采用纯 C# 实现，不依赖 OpenCV 等第三方视觉库。系统专为半导体和精密制造领域的缺陷检测、尺寸测量、定位等应用场景设计。

### 1.1 设计目标

- **零外部依赖**：所有图像处理算法纯 C# 实现，方便部署和学习
- **模块化架构**：每个功能独立为处理器类，通过管线组合
- **可测试性**：ImageData 使用纯字节数组，脱离 WPF 依赖
- **工业级接口**：相机抽象层、标定服务、测量工具一应俱全

## 2. 核心数据结构

### 2.1 ImageData — 纯字节数组图像

```csharp
public class ImageData
{
    public int Width { get; set; }      // 图像宽度（像素）
    public int Height { get; set; }     // 图像高度（像素）
    public int Channels { get; set; }   // 通道数（1=灰度, 3=RGB）
    public int Stride { get; set; }     // 行跨度（字节）
    public byte[] Pixels { get; set; }  // 像素数据

    public static ImageData Create(int width, int height, int channels);
}
```

**设计思路**：
- 不使用 WPF 的 BitmapSource 或 System.Drawing.Bitmap
- 纯字节数组表示，可直接用于算法计算
- Stride = Width × Channels，保证行对齐
- 灰度图像像素寻址：`Pixels[y * Stride + x]`
- RGB 图像像素寻址：`Pixels[y * Stride + x * 3 + channel]`

### 2.2 处理器接口 — IImageProcessor

```csharp
public interface IImageProcessor
{
    string Name { get; }
    ImageData Process(ImageData input, Dictionary<string, object>? parameters = null);
}
```

所有图像处理步骤都实现此接口，可以：
- 单独使用（如 `grayscale.Process(image)`）
- 组合到管线中（如 `pipeline.AddStep(grayscale)`）

## 3. 图像处理算法

### 3.1 灰度化（GrayscaleProcessor）

将多通道图像转换为单通道灰度图像。

**算法**：加权平均法（ITU-R BT.601 标准）
```
Gray = 0.299 × R + 0.587 × G + 0.114 × B
```

人眼对绿色最敏感，因此绿色分量权重最大。

### 3.2 高斯模糊（GaussianBlurProcessor）

使用高斯核对图像进行平滑滤波，抑制高频噪声。

**算法**：
1. 生成 N×N 高斯核（默认 5×5，σ=1.0）
2. 对图像进行二维卷积
3. 边界处理：跳过边界像素

**高斯核计算公式**：
```
G(x, y) = (1 / 2πσ²) × exp(-(x² + y²) / 2σ²)
```

### 3.3 中值滤波（MedianFilterProcessor）

非线性滤波，取邻域中值替代中心像素值。

**特点**：
- 对椒盐噪声效果极佳（优于高斯模糊）
- 保持边缘锐度
- 计算量较大（需要排序）

### 3.4 Sobel 边缘检测（SobelEdgeProcessor）

使用 Sobel 算子检测图像中的边缘。

**算法**：
```
Gx = [-1 0 +1]    Gy = [-1 -2 -1]
     [-2 0 +2]          [ 0  0  0]
     [-1 0 +1]          [+1 +2 +1]

梯度幅值 = √(Gx² + Gy²)
```

分别在 X 和 Y 方向计算梯度，合成为梯度幅值。

### 3.5 阈值二值化

#### 固定阈值（ThresholdProcessor）
- 像素值 ≥ 阈值 → 255（白色）
- 像素值 < 阈值 → 0（黑色）
- 支持反转模式

#### OTSU 自动阈值（OtsuThresholdProcessor）
**算法**：遍历 0~255 所有可能阈值，选择使类间方差最大的阈值。

```
类间方差 σ²(t) = ω₀(t) × ω₁(t) × [μ₀(t) - μ₁(t)]²

其中：
ω₀(t) = 前景像素比例
ω₁(t) = 背景像素比例
μ₀(t) = 前景均值
μ₁(t) = 背景均值
```

**优点**：无需手动设定阈值，自适应图像亮度分布。

### 3.6 形态学运算（MorphologyProcessor）

基于结构元素（Structuring Element）的图像变换。

| 运算 | 公式 | 用途 |
|------|------|------|
| 腐蚀（Erode） | 取局部最小值 | 去除小白点、细化边界 |
| 膨胀（Dilate） | 取局部最大值 | 填充小黑洞、粗化边界 |
| 开运算（Open） | 先腐蚀后膨胀 | 去除小目标、平滑边界 |
| 闭运算（Close） | 先膨胀后腐蚀 | 填充小孔洞、连接断裂 |
| 顶帽（TopHat） | 原图 - 开运算 | 提取明亮小目标 |
| 黑帽（BlackHat）| 闭运算 - 原图 | 提取暗色小目标 |

支持自定义核大小和迭代次数。

## 4. 高级分析算法

### 4.1 模板匹配（TemplateMatcher）

在大图中搜索与模板图像最相似的区域。

#### NCC（归一化互相关）算法

```
NCC(x,y) = Σ[(I(x+i,y+j) - Ī) × (T(i,j) - T̄)] / 
           √[Σ(I(x+i,y+j) - Ī)² × Σ(T(i,j) - T̄)²]
```

**NCC 值范围**：-1.0 到 +1.0
- +1.0 = 完美匹配
- 0.0 = 无相关
- -1.0 = 完全反转

#### 图像金字塔加速

```
原始层 (640×480)  ←─ 精确匹配（仅在候选区域）
        ↑
第1层  (320×240)   ←─ 中间匹配
        ↑
第2层  (160×120)   ←─ 粗搜索（全图扫描）
```

1. 从最顶层（最小分辨率）开始全图搜索
2. 将候选位置映射到下一层，在局部区域精化搜索
3. 逐层精化直到原始分辨率

**加速效果**：3层金字塔可将搜索速度提升约 16 倍。

#### NMS（非最大值抑制）

当匹配结果中有多个重叠区域时，只保留得分最高的。
判断重叠：两个结果的中心距离 < 模板尺寸的一半。

### 4.2 Blob 分析（BlobAnalyzer）

连通域分析 — 识别二值图像中的独立区域。

#### 两遍扫描 + 并查集（Union-Find）算法

**第一遍扫描**：
```
遍历每个前景像素 (值 < 128)：
  - 检查上方和左方邻居
  - 如果都没有标签 → 分配新标签
  - 如果只有一个有标签 → 继承该标签
  - 如果两个都有不同标签 → 取较小标签，记录等价关系
```

**并查集**：
```csharp
// 查找根节点（含路径压缩）
int Find(int x) {
    if (parent[x] != x) 
        parent[x] = Find(parent[x]);  // 路径压缩
    return parent[x];
}

// 合并两个集合
void Union(int a, int b) {
    parent[Find(b)] = Find(a);
}
```

**第二遍扫描**：用 Find() 将所有临时标签替换为最终标签。

#### 特征提取

对每个连通域计算：
- **面积**：像素数量
- **质心**：`(ΣX/N, ΣY/N)`
- **外接矩形**：`(minX, minY, width, height)`
- **周长**：边界像素数量
- **圆度**：`4π × 面积 / 周长²`（1.0 = 完美圆形）
- **伸长率**：外接矩形长轴/短轴比

### 4.3 直方图分析（HistogramAnalyzer）

#### 直方图计算
统计每个灰度值（0~255）出现的次数。

#### OTSU 阈值
见 3.5 节。

#### 直方图均衡化
通过非线性变换扩展图像的灰度范围，增强对比度。

```
步骤：
1. 计算累积分布函数 CDF(k) = Σ h(i)/N, i=0..k
2. 映射：新值 = round(CDF(旧值) × 255)
```

## 5. 测量工具

### 5.1 距离测量
```
d = √[(x₂-x₁)² + (y₂-y₁)²]
```

### 5.2 角度测量
三点角度（P2 为顶点）：
```
cos θ = (P1-P2)·(P3-P2) / (|P1-P2| × |P3-P2|)
```

### 5.3 圆拟合 — Kasa 算法

最小二乘法拟合圆心和半径。

**原理**：将圆方程 `(x-a)² + (y-b)² = r²` 展开并线性化：
```
x² + y² = 2ax + 2by + (r² - a² - b²)
```

令 c = r² - a² - b²，构建超定方程组：
```
[2x₁  2y₁  1] [a]   [x₁²+y₁²]
[2x₂  2y₂  1] [b] = [x₂²+y₂²]
[...  ...  .] [c]   [...     ]
```

用最小二乘法（AᵀA · x = Aᵀb）求解。

### 5.4 直线拟合

最小二乘法拟合直线 y = kx + b。

```
k = (NΣxy - ΣxΣy) / (NΣx² - (Σx)²)
b = (Σy - kΣx) / N
```

拟合误差 = 各点到直线的均方根距离。

## 6. 标定服务

### 6.1 9 点仿射标定

通过已知的像素坐标→物理坐标对应关系，求解仿射变换矩阵。

**仿射变换**：
```
[wx]   [a₀₀  a₀₁  a₀₂]   [px]
[wy] = [a₁₀  a₁₁  a₁₂] × [py]
                            [ 1]
```

需要至少 3 对已知点，推荐使用 9 点（3×3 网格）以获得更好的精度。

**求解方法**：
构建超定方程组，用高斯消元法求解 6 个未知系数。

### 6.2 标定精度评估

- **平均重投影误差**：将标定点代入变换矩阵，计算预测值与真实值的平均偏差
- **最大重投影误差**：最大偏差值
- 通常要求平均误差 < 0.5 像素

## 8. 图像处理管线

### 8.1 VisionPipeline

将多个处理步骤串联为可配置的处理链。

```csharp
var pipeline = new VisionPipeline();
pipeline.AddStep(new GrayscaleProcessor());
pipeline.AddStep(new GaussianBlurProcessor());
pipeline.AddStep(new OtsuThresholdProcessor());

// 执行管线
ImageData result = pipeline.Execute(inputImage);

// 执行管线（带诊断）
List<PipelineStepResult> diagnostics = pipeline.ExecuteWithDiagnostics(inputImage);
foreach (var step in diagnostics)
{
    Console.WriteLine($"{step.StepName}: {step.Duration.TotalMilliseconds:F1}ms");
}
```

### 8.2 默认检测管线

```
灰度化 → 高斯模糊 → OTSU 阈值 → 开运算（形态学）
```

适用于大多数缺陷检测场景。

## 9. 完整使用示例

```csharp
// 创建视觉引擎
var engine = new VisionEngineV2();

// 连接模拟相机
await engine.ConnectCameraAsync();

// 采集图像
var image = await engine.CaptureAsync();

// 图像处理
var gray = engine.ToGrayscale(image);
var blurred = engine.GaussianBlur(gray);
var binary = engine.OtsuThreshold(blurred);
var cleaned = engine.Morphology(binary, MorphologyOperation.Open);

// Blob 分析
var blobs = engine.FindBlobs(cleaned);
foreach (var blob in blobs)
{
    Console.WriteLine($"Blob: 面积={blob.Area}, 中心=({blob.CenterX:F1},{blob.CenterY:F1}), " +
                      $"圆度={blob.Circularity:F3}");
}

// 圆拟合
var circlePoints = blobs.Where(b => b.Circularity > 0.8)
    .Select(b => (b.CenterX, b.CenterY)).ToList();
if (circlePoints.Count >= 3)
{
    var (cx, cy, r, err) = engine.FitCircle(circlePoints);
    Console.WriteLine($"拟合圆：中心=({cx:F1},{cy:F1}), 半径={r:F1}, 误差={err:F3}");
}

// 测量
double dist = engine.MeasureDistance(0, 0, 100, 100);
Console.WriteLine($"距离={dist:F1} px");

// 标定
var calibPoints = new List<CalibrationPointPair>
{
    new() { PixelX = 0, PixelY = 0, WorldX = 0, WorldY = 0 },
    new() { PixelX = 100, PixelY = 0, WorldX = 10, WorldY = 0 },
    new() { PixelX = 0, PixelY = 100, WorldX = 0, WorldY = 10 },
    // ... 更多标定点
};
var calibData = engine.Calibrate(calibPoints);
var (wx, wy) = engine.PixelToWorld(50, 50);
Console.WriteLine($"像素(50,50) → 物理({wx:F2},{wy:F2})");

engine.Dispose();
```
