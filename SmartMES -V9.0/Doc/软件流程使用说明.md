# SmartMES V9.0 软件流程使用说明

## 1. 系统启动流程

### 1.1 启动序列
```
应用程序启动
    │
    ├── 1. 加载系统配置（appsettings.json）
    │
    ├── 2. 初始化核心服务
    │       ├── LoggingService（日志服务）
    │       ├── EventBusService（事件总线）
    │       ├── AlarmService（报警服务）
    │       └── ConfigurationCenter（配置中心）
    │
    ├── 3. 初始化通信服务
    │       ├── TcpCommunicationService
    │       ├── SerialCommunicationService
    │       ├── ModbusTcpService
    │       └── SecsGemService（SECS/GEM）
    │
    ├── 4. 初始化功能模块
    │       ├── MotionControl（运动控制）
    │       ├── VisionEngine（视觉引擎）
    │       ├── DatabaseModule（数据库）
    │       └── MesCommModule（MES通信）
    │
    ├── 5. 初始化 UI 层
    │       ├── MainWindow
    │       ├── ViewModels
    │       └── 导航框架
    │
    └── 6. 显示登录界面 → 进入主界面
```

### 1.2 服务依赖关系
```
ILoggingService ← 所有服务都依赖日志
IEventBus ← 模块间通信的核心
IAlarmService ← 依赖 IEventBus
ICommunicationService ← 依赖 ILoggingService
MotionControl ← 依赖 ILoggingService, IEventBus
VisionEngine ← 独立（纯算法）
SecsGemService ← 依赖 ILoggingService, IEventBus, IAlarmService
```

## 2. 生产工作流程

### 2.1 标准生产流程
```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  准备   │───→│  生产   │───→│  检测   │───→│  完成   │
│  阶段   │    │  阶段   │    │  阶段   │    │  阶段   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │
     ▼              ▼              ▼              ▼
 · 设备连接     · 运动执行     · 视觉采集     · 数据保存
 · 回零操作     · G代码运行    · 缺陷检测     · 报表生成
 · 配方加载     · 状态监控     · OK/NG判定    · MES上报
 · 安全检查     · 报警处理     · 分拣动作     · 日志归档
```

### 2.2 详细步骤

#### 步骤1：准备阶段
1. **设备连接**
   - 连接 PLC（Modbus TCP）
   - 连接运动控制卡（仿真模式自动就绪）
   - 连接相机（仿真模式自动就绪）
   - 连接 SECS/GEM 主机（半导体场景）

2. **回零操作**
   - 执行全轴回零（G28）
   - 等待所有轴回零完成
   - 确认各轴位置为零点

3. **配方加载**
   - 选择生产配方
   - 加载运动参数（速度、加速度、坐标系偏移）
   - 加载视觉参数（阈值、检测区域、模板）
   - 加载工艺参数

4. **安全检查**
   - 检查安全门状态
   - 检查急停按钮状态
   - 检查气压/真空状态
   - 所有安全条件满足后允许启动

#### 步骤2：生产阶段
1. **运动执行**
   - 按 G 代码程序执行运动
   - 多轴协同运动（插补）
   - 实时监控各轴位置和状态

2. **状态监控**
   - 监控运动状态（Idle/Running/Paused/Error）
   - 监控通信状态
   - 监控报警状态

3. **异常处理**
   - 报警触发 → 暂停生产 → 处理异常 → 恢复生产
   - 急停触发 → 所有轴停止 → 排除故障 → 复位 → 重新启动

#### 步骤3：检测阶段
1. **视觉采集**
   - 运动到拍照位置
   - 触发相机采集
   - 获取图像数据

2. **缺陷检测**
   - 执行视觉处理管线
   - 分析检测结果
   - 判定 OK/NG

3. **分拣动作**
   - OK 产品 → 移动到合格品位置
   - NG 产品 → 移动到废品箱位置

#### 步骤4：完成阶段
1. **数据保存**
   - 保存生产数据到数据库
   - 记录追溯信息

2. **报表生成**
   - 生成生产报表（产量、良率、节拍）
   - 导出数据文件

3. **MES 上报**
   - 通过 HTTP/MQTT 上报生产结果
   - SECS/GEM 事件上报（S6F11）

## 3. SECS/GEM 通信流程

### 3.1 连接建立流程
```
设备端                              主机端
  │                                   │
  │──── TCP Connect ─────────────────→│  ① TCP 连接
  │                                   │
  │──── Select.req ──────────────────→│  ② HSMS 选择请求
  │←─── Select.rsp ──────────────────│
  │                                   │
  │──── S1F13 (Establish Comm) ─────→│  ③ 建立通信
  │←─── S1F14 (COMMACK=0) ──────────│
  │                                   │
  │──── S1F17 (Request Online) ─────→│  ④ 请求上线
  │←─── S1F18 (ONLACK=0) ──────────│
  │                                   │
  │     *** 正常通信状态 ***           │
```

### 3.2 正常通信流程
```
设备端                              主机端
  │                                   │
  │←─── S1F1 (Are You There) ────────│  心跳检测
  │──── S1F2 (I Am Here) ──────────→│
  │                                   │
  │──── S6F11 (Event Report) ───────→│  事件上报
  │←─── S6F12 (ACK) ────────────────│
  │                                   │
  │←─── S2F41 (Remote Command) ─────│  远程命令
  │──── S2F42 (HCACK) ─────────────→│
  │                                   │
  │←─── S1F3 (SV Request) ──────────│  状态查询
  │──── S1F4 (SV Data) ────────────→│
  │                                   │
  │──── S5F1 (Alarm Set) ──────────→│  告警上报
  │←─── S5F2 (ACK) ─────────────────│
```

### 3.3 GEM 状态机
```
通信状态模型：
  Disabled ──Enable──→ WaitCommunicating ──S1F13/14──→ Communicating
                                                          │
                                                    通信失败
                                                          │
                                                          ▼
                                                   NotCommunicating

控制状态模型（仅在 Communicating 状态下有效）：
  EquipmentOffline ──请求上线──→ AttemptOnline
                                      │
                              ┌───────┴───────┐
                              ▼               ▼
                         OnlineLocal    OnlineRemote
                              │               │
                              └───────┬───────┘
                                      │
                                  请求离线
                                      │
                                      ▼
                              EquipmentOffline
```

## 4. 运动控制流程

### 4.1 单轴运动流程
```
Idle ──MoveTo(target)──→ Running ──到达目标──→ Idle
  │                         │
  │                    Pause()
  │                         │
  │                         ▼
  │                      Paused ──Resume()──→ Running
  │
  ├──Home()──→ Homing ──回零完成──→ Idle
  │
  └──SetError()──→ Error ──Reset()──→ Idle
```

### 4.2 G 代码执行流程
```
加载 G 代码程序
    │
    ├── 逐行解析 G 代码
    │       │
    │       ├── G0/G1：直线运动
    │       │     └── 坐标变换（G54~G59）
    │       │     └── 绝对/增量模式（G90/G91）
    │       │
    │       ├── G2/G3：圆弧插补
    │       │     └── 分解为微线段
    │       │     └── 前瞻速度规划
    │       │
    │       ├── G4：暂停等待
    │       │
    │       ├── G28：回零
    │       │     └── 选择回零策略
    │       │
    │       └── M2/M30：程序结束
    │
    └── 返回执行结果
```

### 4.3 多工位协同流程
```
调度器启动
    │
    ├── 工位1（独立循环）          ├── 工位2（独立循环）          ├── 工位3（独立循环）
    │   ├── 定位（XYZ运动）       │   ├── 定位                  │   ├── 定位
    │   ├── 拍照（相机采集）       │   ├── 拍照                  │   ├── 拍照
    │   ├── 检测（视觉分析）       │   ├── 检测                  │   ├── 检测
    │   ├── 分拣（OK/NG动作）     │   ├── 分拣                  │   ├── 分拣
    │   └── 复位                  │   └── 复位                  │   └── 复位
    │                             │                             │
    └─────────────────── 所有工位完成 → 生产结束
```

## 5. 视觉检测流程

### 5.1 标准检测流程
```
图像采集
    │
    ▼
预处理（灰度化 → 滤波 → 增强）
    │
    ▼
特征提取（阈值分割 / 边缘检测 / 模板匹配）
    │
    ▼
缺陷分析（Blob分析 → 特征计算 → 分类）
    │
    ▼
结果判定（OK/NG → 标注 → 输出）
```

### 5.2 视觉处理管线配置示例

#### 表面缺陷检测
```
灰度化 → 高斯模糊(σ=1.5) → OTSU阈值 → 形态学开运算(3×3) → Blob分析(minArea=50)
```

#### 模板定位
```
灰度化 → 高斯模糊(σ=1.0) → 模板匹配(NCC, minScore=0.85)
```

#### 尺寸测量
```
灰度化 → Sobel边缘 → 阈值(128) → 边缘点检测 → 直线拟合 → 距离计算
```

### 5.3 标定流程
```
准备标定板（9个标定点）
    │
    ▼
依次移动到各标定点位置
    │
    ▼
采集图像，记录像素坐标
    │
    ▼
输入对应的世界坐标（mm）
    │
    ▼
最小二乘法求解仿射矩阵
    │
    ▼
计算标定误差（应 < 0.5 像素）
    │
    ▼
保存标定数据
```

## 6. 异常处理流程

### 6.1 报警处理
```
异常发生
    │
    ▼
报警触发（AlarmService.TriggerAlarm）
    │
    ├── 事件总线广播（IEventBus.Publish）
    │
    ├── UI 显示报警（红色提示）
    │
    ├── SECS/GEM 上报（S5F1）
    │
    └── 日志记录
    │
    ▼
操作员处理
    │
    ├── 查看报警详情
    ├── 排除故障
    ├── 确认报警
    └── 恢复运行
```

### 6.2 急停处理
```
急停触发
    │
    ├── 所有轴立即停止（EmergencyStop）
    ├── 所有输出关闭
    ├── 报警记录
    └── 状态切换为 Error
    │
    ▼
故障排除
    │
    ▼
复位操作
    │
    ├── 确认安全条件
    ├── 全轴复位（ResetAll）
    ├── 全轴回零（HomeAllAsync）
    └── 恢复到 Idle 状态
```

### 6.3 通信中断处理
```
通信中断检测
    │
    ├── 触发 ConnectionChanged 事件
    ├── 报警记录
    │
    ▼
自动重连（如果配置了自动重连）
    │
    ├── 等待 T5 超时
    ├── 尝试重新连接
    ├── 重新握手
    └── 恢复通信
```

## 7. 数据流向

### 7.1 实时数据流
```
硬件/仿真层
    │
    ▼
通信服务层（TCP/Modbus/SECS-GEM）
    │
    ▼
事件总线（IEventBus）
    │
    ├──→ ViewModel（UI 更新）
    ├──→ 日志服务（记录）
    ├──→ 报警服务（监控）
    └──→ 数据库（存储）
```

### 7.2 命令流向
```
UI 操作
    │
    ▼
ViewModel（命令处理）
    │
    ▼
服务/模块层（业务逻辑）
    │
    ▼
通信服务层（协议封装）
    │
    ▼
硬件/仿真层（执行）
```
