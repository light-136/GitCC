# SmartMES V9.0 — 30轴工业级运动控制系统设计文档

## 1. 概述

### 1.1 系统目标

30轴工业级运动控制系统是 SmartMES V9.0 的高级运动控制扩展，面向半导体设备、精密加工中心等大型工业设备的多轴协调控制需求。

**核心特性**：
- **30轴并行管理**：6个轴组，覆盖龙门、机械臂、传送带、主轴、辅助轴
- **多通道并行执行**：6个独立 G 代码通道，支持多工位同时加工
- **碰撞检测与安全互锁**：后台实时监控轴间碰撞风险
- **性能监控与诊断**：跟随误差、周期时间、负载率实时采样
- **纯 C# 实现**：完全仿真运行，便于学习和理解

### 1.2 适用场景

| 场景 | 说明 |
|------|------|
| 双龙门加工 | 主副龙门各自加工不同区域，需碰撞检测 |
| 机械臂搬运 | 6关节机械臂在工位间搬运物料 |
| 多工位装配 | 传送带+定位轴+加工轴协调 |
| 主轴加工 | 多主轴同时旋转加工 |

## 3. 核心数据模型

```csharp

{
    public string Name { get; set; }              // 轴名称（唯一标识）

    public double MaxAcceleration { get; set; }    // 最大加速度
    public double MaxJerk { get; set; }            // 最大加加速度

    public string GroupName { get; set; }           // 所属轴组
    public int ChannelId { get; set; }             // 所属通道

}
```

## 4. 子模块设计

### 4.1 轴组管理器（AxisGroupManager）

**功能**：将30轴组织为逻辑组，提供组级操作。

**核心API**：

| 方法 | 功能 |
|------|------|
| `RegisterAxis(axis)` | 注册轴 |
| `CreateGroup(definition)` | 创建轴组 |
| `GetGroupAxes(groupName)` | 获取组内轴列表 |
| `AreAllAxesIdle(groupName)` | 检查组内轴是否全部空闲 |
| `ConfigureGantry(config)` | 配置龙门双驱同步 |
| `CheckGantryDeviation(master)` | 检测龙门偏差 |
| `EmergencyStopGroup(groupName)` | 轴组急停 |

**龙门双驱同步原理**：

```
主轴 X1 ────┐
             ├── 运动指令 ──→ 两轴同步运动
从轴 X1' ───┘
             
偏差监控：|X1.Position - X1'.Position| < MaxDeviation
```

### 4.2 多通道控制器（MultiChannelController）

**功能**：支持多个独立 G 代码程序并行执行。

**通道状态机**：

```
         StartChannel
Idle ──────────────→ Running
  ↑                    │ │
  │    StopChannel     │ │ PauseChannel
  ←────────────────────┘ │
  ↑                      ↓
  │    ResumeChannel   Paused
  ←────────────────────── 
  ↑
  │   Error/Exception
Error ←──────── Running
```

**同步屏障（SyncBarrier）**：

当多通道需要在某个点同步时：

```
通道0: G1 X100 → [Barrier #1] → G1 X200
                      ↕
通道1: G1 Y100 → [Barrier #1] → G1 Y200
```

两个通道都到达 Barrier #1 后才继续执行后续指令。

### 4.3 碰撞检测器（CollisionDetector）

**检测原理**：

```
      轴1位置范围
      ←───────────→
   Axis1Min     Axis1Max
      ┌───────────┐
      │  危 险 区  │  ← 两轴同时在此范围内 → 碰撞告警
      └───────────┘
   Axis2Min     Axis2Max
      ←───────────→
      轴2位置范围
```

**判定逻辑**：

```
if (Axis1Min ≤ axis1.Position ≤ Axis1Max) AND
   (Axis2Min ≤ axis2.Position ≤ Axis2Max):
   
   distance = |axis1.Position - axis2.Position|
   
   if distance < MinSafeDistance:
       → "Critical" — 立即停止两轴
   else:
       → "Warning" — 记录事件
```

**后台监控**：50ms 周期扫描所有碰撞区域，"Critical" 级别自动触发 `axis.Stop()`。

### 4.4 性能监控器（PerformanceMonitor）

**采样指标**（每100ms采样一次）：

| 指标 | 计算方式 |
|------|---------|
| 跟随误差 | `TargetPosition - Position` |
| 当前速度 | `axis.Velocity` |
| 电机负载 | `min(100, |Velocity| / MaxVelocity * 50)` (仿真) |
| 周期时间 | 实测单次扫描耗时（Stopwatch） |

**性能报告**（SystemPerformanceReport）：

```
总轴数: 30
运动中: 4
报错: 0
最大跟随误差: 0.015 mm
平均周期时间: 2.3 ms
碰撞告警: 0
```

## 5. ThirtyAxisController 编排器

### 5.1 初始化流程

```
1. 创建 MultiAxisController → 添加30个 AxisController
2. 将每个轴注册到: GroupManager, ChannelController, 
   CollisionDetector, PerformanceMonitor
3. 按 GroupName 自动创建6个轴组
4. 按 ChannelId 自动创建6个通道
5. 配置默认碰撞区域（主副龙门X轴互锁）
```

### 5.2 典型工作流程

```
初始化
  ↓
回零 → HomeAllAxesAsync() 或 HomeGroupAsync("Gantry1")
  ↓
加载程序 → 多通道各自准备 G 代码
  ↓
运行 → RunMultiChannelAsync(programs)
  │     ├─ 自动启动碰撞监控
  │     ├─ 自动启动性能采样
  │     └─ 多通道并行执行
  ↓
停止 → 程序结束 / EmergencyStop()
  ↓
报告 → GetPerformanceReport()
```

### 5.3 软限位保护

```csharp
public bool MoveAxis(string axisName, double target)
{
    // 检查目标位置是否在软限位范围内
    if (target > cfg.SoftLimitPositive || target < cfg.SoftLimitNegative)
        return false; // 拒绝运动
    
    return axis.MoveTo(target);
}
```

## 6. 使用示例

```csharp
// 创建30轴控制器
var controller = new ThirtyAxisController();

// 订阅日志
controller.MessageLogged += (_, msg) => Console.WriteLine(msg);

// 全轴回零
await controller.HomeAllAxesAsync();

// 单轴运动（含软限位保护）
bool ok = controller.MoveAxis("X1", 100.0);  // → true
bool bad = controller.MoveAxis("X1", 999.0); // → false（超出限位）

// 查看轴组状态
var status = controller.GetGroupStatus("Robot");
foreach (var (name, (state, pos)) in status)
    Console.WriteLine($"{name}: {state}, 位置={pos:F2}mm");

// 双龙门并行加工
var programs = ThirtyAxisController.GetDualGantrySamplePrograms();
await controller.RunMultiChannelAsync(programs);

// 获取性能报告
var report = controller.GetPerformanceReport(60);
Console.WriteLine($"总轴数: {report.TotalAxes}");
Console.WriteLine($"最大跟随误差: {report.MaxFollowingError:F3}mm");

// 急停
controller.EmergencyStop();

// 释放资源
controller.Dispose();
```

## 8. 扩展指南

### 8.1 添加新轴

在 `ThirtyAxisController` 的 `DefaultAxisConfigs` 数组中添加新条目：

```csharp
new AxisConfig
{
    Name = "X3",
    AxisType = AxisType.Linear,
    MaxVelocity = 500,
    MaxAcceleration = 2000,
    GroupName = "Gantry3",  // 新组或已有组
    ChannelId = 6           // 新通道或已有通道
}
```

### 8.2 添加碰撞区域

```csharp
controller.CollisionDetector.AddZone(new CollisionZone
{
    Name = "Robot_Conveyor_Collision",
    Axis1Name = "J1",
    Axis2Name = "C1",
    Axis1Min = -90, Axis1Max = 90,
    Axis2Min = 100, Axis2Max = 300,
    MinSafeDistance = 50.0
});
```

### 8.3 接入真实伺服驱动

替换 `AxisController` 的运动仿真为实际伺服通信：

1. 创建 `IServoDriver` 接口
2. 在 `AxisController.MoveToPosition()` 中调用驱动 API
3. 将位置反馈替换为编码器读数
4. 将碰撞检测周期调整为实时总线周期（如 EtherCAT 1ms）
