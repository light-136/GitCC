# SmartMES 完善版工业上位机完整设计文档

> 版本：v4.0 设计版  
> 基于项目：`SmartMES`  
> 目标：形成一份可用于汇报、答辩、二次开发、实施交接的完整工业上位机设计文档。

---

## 1. 项目定义

`SmartMES` 是一个基于 `WPF + .NET 9 + MVVM` 构建的工业上位机系统，面向智能制造现场，承担设备监控、流程执行、MES 数据交换、质量检测、追溯、安全互锁、参数与权限管理等职责。

它既是一个完整的桌面工业执行端，也是一个可继续扩展为真实项目的基础平台。

### 1.1 项目目标

1. 提供统一的人机交互入口；
2. 集成制造现场常见能力模块；
3. 支持设备、工艺、质量、数据的一体化展示与处理；
4. 具备工程化基础：权限、日志、异常、测试、扩展；
5. 为真实现场接入预留架构边界。

### 1.2 适用场景

- 设备上位机
- 工站执行终端
- 智能制造演示平台
- 生产过程采集与追溯终端
- 工业项目原型系统

## 2. 总体架构设计

### 2.1 分层结构

```text
SmartMES.UI          表现层
SmartMES.Services    服务层
SmartMES.Modules     业务模块层
SmartMES.Core        核心基础层
SmartMES.Native      原生扩展层
SmartMES.Tests       测试层
```

### 2.2 各层职责

#### `SmartMES.UI`
- WPF 页面展示
- 用户交互
- 菜单导航
- 状态可视化
- 权限驱动显示

#### `SmartMES.Services`
- 日志服务
- 报警服务
- 事件总线
- 通信服务
- 自动化流程服务
- 参数服务

#### `SmartMES.Modules`
- 数据库模块
- 文件处理模块
- MES 通信模块
- 运动控制模块
- 视觉模块
- 视觉运动协同模块
- Native 互操作模块

#### `SmartMES.Core`
- MVVM 基础设施
- 接口与模型
- 状态机
- 调度器
- IO 抽象
- 配方系统
- 安全互锁
- 追溯系统
- 插件机制

#### `SmartMES.Native`
- C++ 原生动态库
- 数学与矩阵计算

#### `SmartMES.Tests`
- 状态机测试
- 调度测试
- 运动控制测试
- 权限测试
- 配方/安全/追溯测试

### 2.3 设计原则

1. 分层清晰；
2. 模块解耦；
3. 工业安全优先；
4. 状态可观测；
5. 行为可追溯；
6. 逻辑可测试；

8. 后续可扩展。

## 10. 线程与运行机制设计

### 10.1 原则
- UI 线程仅负责显示与轻量协调；
- 后台任务通过异步或工作线程执行；
- 跨线程 UI 更新必须通过 Dispatcher。

### 10.2 当前机制
- `DispatcherTimer`
- `Task.Run`
- `async/await`
- 后台线程驱动轴运动模拟

### 10.3 风险控制
- 禁止后台线程直接操作 UI；
- 长时动作支持停止和取消；
- 共享数据结构必须线程安全。

## 13. 典型业务流程

### 13.1 登录与权限流程
登录 → 验证角色 → 发布登录事件 → 刷新导航 → 越权页面回退 → 显示用户状态

### 13.2 数据库统计流程
进入数据库页面 → 连接数据库 → 初始化/读取数据 → 执行统计 → 展示结果 → 导出

### 13.3 运动控制流程
进入运动页面 → 查看轴状态 → 发出控制命令 → 状态机切换 → UI 刷新 → 异常联动安全与报警

### 13.4 视觉检测流程
采集/生成图像 → 调整参数 → 执行检测 → 输出 OK/NG → 标注缺陷 → 记录或上报

### 13.5 追溯流程
创建产品追溯 → 工序开始 → 记录参数 → 工序结束 → 写入结果 → 形成完整追溯链

### 13.6 MES 上报流程
建立连接 → 获取工单 → 执行生产 → 采集结果 → 结果上报 → 记录日志
