# SmartMES 单元测试设计文档

> 版本: v1.0  
> 日期: 2026-04-02  
> 范围: `SmartMES.Core` + `SmartMES.Modules` 首批单元测试

---

## 1. 设计目标

建立可持续的自动化测试基线，优先覆盖“纯逻辑、高复用、对系统稳定性影响大”的核心组件：

1. 状态机引擎（流程正确性）；
2. 运动控制基础对象（状态转换与移动行为）；
3. G代码解析器（输入解释正确性）。

## 3. 测试分层与策略

### 3.1 核心状态机层（StateMachine）

文件：`StateMachineEngineTests.cs`

覆盖点：

1. 标准状态机初始态应为 `Idle`；
2. 标准触发链路：`Idle -> Running -> Paused -> Running -> Idle`；
3. 非法触发应返回 `false` 且不改变状态；
4. Guard 条件应生效（先拦截后放行）；
5. 强制状态切换（`ForceState`）应生效。

设计理由：

- 状态机是多个业务模块的行为基石，错误会放大到整机流程。

### 3.2 运动轴行为层（AxisController）

文件：`AxisControllerTests.cs`

覆盖点：

1. `MoveTo` 能完成移动并回到 `Idle`；
2. `Pause/Resume` 状态切换有效，且可最终完成移动；
3. `SetError` + `Reset` 可以从 `Error` 恢复到 `Idle`。

设计理由：

- 运动轴属于高风险能力，需优先验证状态切换与最终到位能力。

### 3.3 G代码解析层（GCodeCommand）

文件：`GCodeCommandTests.cs`

覆盖点：

1. `G1` 解析轴坐标与进给速度；
2. `M2` 程序结束解析；
3. 未知指令解析（`Unknown`）；
4. 空行解析行为（保持 `Unknown` + 空轴参数）。

设计理由：

- G代码是10轴模块的输入边界，解析错误会导致执行偏差。

---

## 4. 关键设计决策

1. **先测逻辑，不测UI渲染**：WPF渲染属于集成/界面测试范畴，单元测试阶段聚焦业务逻辑；
2. **控制异步等待时间**：Axis相关测试使用 `WaitUntilAsync`，避免死等；
3. **修正输入默认行为**：通过测试发现并修复 `GCodeCommand.Type` 默认值问题（默认应为 `Unknown`）。
