# SmartMES 详细设计文档

> 版本: v3.1.0 | 日期: 2026-04-02 | 范围: UI导航、运动控制中心、10轴版面切换、工业级模块集成

---

## 1. 设计目标

SmartMES 是基于 `WPF + .NET 9` 的工业制造执行系统上位机。当前版本重点目标：

1. 建立稳定的分层架构与可维护的 MVVM 页面组织；
2. 将“运动控制中心（3轴）”与“10轴版面切换（G代码）”设计为可直接演示、可扩展、可复用的能力模块；
3. 通过统一日志、统一异常处理、统一导航机制，降低现场排障成本；
4. 形成可持续迭代的注释与文档规范。

---

## 2. 分层架构设计

```text
SmartMES.UI          表现层（WPF + ViewModel）
  └─ SmartMES.Services  服务层（日志、通信、自动化、设备）
      └─ SmartMES.Modules   业务模块层（运动、视觉、数据库、文件、MES）
          └─ SmartMES.Core  核心基础层（调度、状态机、IO、配方、安全、追溯、插件）
```

### 2.1 分层职责

- `SmartMES.UI`
  - 仅处理交互与展示，不直接写业务算法；
  - 通过 `RelayCommand + Binding` 驱动页面；
  - 通过 `DataTemplate` 实现 ViewModel 到 View 的自动路由。
- `SmartMES.Services`
  - 提供系统级能力（日志、事件总线、通信、自动化流程）；
  - 向 UI 暴露可组合的服务对象。
- `SmartMES.Modules`
  - 实现业务逻辑（运动控制、视觉、文件、数据库、MES通信）；
  - 与 UI 解耦，支持独立测试。
- `SmartMES.Core`
  - 放置通用基础能力（状态机、调度、IO仿真、配方、安全互锁、追溯、插件加载）。

## 3. UI 导航设计

### 3.1 导航中心

`MainViewModel` 作为主导航协调器，承担：

1. 管理页面 VM 实例生命周期；
2. 执行导航命令并维护 `CurrentPage` 与 `CurrentPageTitle`；
3. 同步顶部状态（当前用户、角色、系统时间）；
4. 保持“菜单名称 / 页面标题 / 功能语义”一致。

### 3.2 当前关键导航标题（已对齐）

- `运动控制中心` -> 对应三轴控制页面；
- `10轴版面切换` -> 对应10轴页面并支持布局切换。

---

## 4. 运动控制中心（3轴）设计

### 4.1 模块定位

`MotionViewModel` 是三轴控制中心（X/Y/Z），用于演示典型设备轴控制流程。

### 4.2 功能边界

- 全局控制：全部回零、随机移动、暂停/恢复、复位、急停；
- 参数控制：全局速度、全局加速度；
- 协同控制：线性插补（多轴联动）；
- 工位控制：上料位 / 加工位 / 安全位预设；
- 状态可视化：轴状态、当前位置、目标位置、日志。

### 4.3 线程与刷新设计

- 每个轴由 `AxisController` 独立状态机驱动；
- UI 通过 `DispatcherTimer(100ms)` 刷新状态；
- 所有 UI 集合写操作均在 Dispatcher 线程执行。

## 5. 10轴版面切换（G代码）设计

### 5.1 模块定位

`Motion10AxisViewModel` 面向“程序执行 + 10轴状态监控 + 布局切换”的综合演示。

### 5.2 功能边界

- 程序控制：单次执行 / 循环执行；
- 程序内容：支持 `G0/G1/G28/M0/M2`；
- 实时统计：执行行数、运行时间、状态文本；
- 安全控制：急停、复位、全部回零；
- 版面切换：
  - `5×2 横向布局`（宽屏）
  - `2×5 纵向布局`（窄屏/纵向监控）

### 5.3 版面切换实现要点

- 页面采用 `UniformGrid` 展示 10 轴卡片；
- `Columns/Rows` 由 ViewModel 属性 `AxisPanelColumns`/`AxisPanelRows` 驱动；
- 切换命令：`Layout52Command`、`Layout25Command`；
- 布局状态通过 `LayoutModeText` 在右上角实时显示。

## 6. 代码注释与维护规范

### 6.1 注释分层规范

- **类注释（必需）**：说明设计职责，不写实现细枝末节；
- **关键属性注释（建议）**：说明 UI 绑定用途、范围约束；
- **关键方法注释（必需）**：描述流程目标和副作用（例如命令状态刷新、急停行为）；
- **复杂逻辑块注释（按需）**：说明为何这么做，不重复“代码表面含义”。

### 6.2 注释质量要求

- 面向“维护者下一次阅读”而不是“当前作者记忆”；
- 与实际代码保持一致，修改行为时同步更新注释；
- 避免空洞描述（如“执行方法”“设置值”）。

## 7. 异常与日志设计

### 7.1 全局异常防护

`App.xaml.cs` 采用三层保护：

1. `DispatcherUnhandledException`（UI线程）；
2. `TaskScheduler.UnobservedTaskException`（任务线程）；
3. `AppDomain.CurrentDomain.UnhandledException`（兜底）。

### 7.2 日志策略

- 页面日志统一采用时间戳前缀；
- 全局日志通过 `ILoggingService` 写入；
- 运动与程序执行日志保留关键信息（状态、行数、异常消息）。

---

## 8. 关键可维护性约束

1. 新增页面必须同时更新：
   - `MainViewModel` 导航命令与标题；
   - `MainWindow.xaml` DataTemplate；
   - 菜单入口与权限开关（若需权限控制）。
2. 涉及线程的 UI 更新统一走 Dispatcher；
3. 运动控制相关 Brush/状态对象必须保持线程安全策略；
4. 文档与代码版本保持同步，功能变更需更新本设计文档。
