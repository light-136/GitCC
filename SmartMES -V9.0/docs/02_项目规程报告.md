# SmartMES 项目开发报告

> 版本: v3.1.0  
> 日期: 2026-04-02  
> 覆盖范围: 架构实现、功能交付、问题修复、测试与后续计划

---

## 1. 项目目标与定位

SmartMES 是面向制造现场的 WPF 上位机系统，目标是提供：

1. 统一监控与操作入口；
2. 可扩展的工业业务模块（运动、视觉、MES、数据库、文件）；
3. 可维护、可测试的分层架构（UI/Services/Modules/Core）；
4. 可落地的安全与追溯能力（互锁、急停、工序链）。

---

## 2. 开发里程碑

### 里程碑 M1：基础监控能力

- 仪表盘、设备管理、通信监控、报警管理、系统日志、自动化流程。

### 里程碑 M2：扩展业务能力

- MES通信、文件处理、数据库、3轴运动控制、Native互操作。

### 里程碑 M3：样例与权限体系

- 10轴G代码执行、视觉检测、视觉+运动协同、用户与权限管理。

### 里程碑 M4：工业基础能力强化

- 调度系统、状态机、IO仿真、配方、互锁安全、追溯、插件机制。

### 里程碑 M5：工程化完善

- 核心注释补强、设计文档重构、快速启动指南更新、单元测试体系建立。

## 3. 当前功能交付清单（按项目层级）

### 3.1 UI层（`SmartMES.UI`）

- 主导航、页面路由、菜单分组、标题同步；
- 关键页面：运动控制中心、10轴版面切换、视觉检测、数据库、文件处理、用户权限。

### 3.2 Services层（`SmartMES.Services`）

- 日志服务、配置服务、事件总线；
- TCP/串口通信模拟；
- 自动化流程引擎、设备服务、报警服务。

### 3.3 Modules层（`SmartMES.Modules`）

- 运动控制（单轴/多轴/10轴+G代码）；
- 视觉引擎（灰度、Sobel、缺陷检测）；
- 文件处理（TXT/CSV/JSON/XML/Excel）；
- 数据库模块、MES通信模块、视觉运动协同模块。

### 3.4 Core层（`SmartMES.Core`）

- MVVM基础设施（命令、属性通知）；
- 状态机、调度器、安全互锁、配方、追溯、IO仿真、插件加载。

---

## 4. 关键修复与质量改进

1. 修复跨线程 UI 资源访问导致的崩溃（Brush冻结 + UI线程刷新）；
2. 修复按钮可执行状态未刷新问题（统一触发 `RaiseCanExecuteChanged`）；
3. 修复 XAML 绑定风险点（避免高风险绑定方式）；
4. 修复 G代码空行默认类型误判（`GCodeType` 默认值改为 `Unknown`）；
5. 统一文档与页面命名：`运动控制中心`、`10轴版面切换`。

## 8. v3.1.1 迭代补充（2026-04-09）

### 本轮完成

1. 将用户登录事件扩展为携带 `PagePermissions`，打通“登录态 -> 主导航权限”链路；
2. `MainViewModel` 新增页面访问判定属性，并在登录/登出后实时刷新左侧菜单显示；
3. 当当前页面对当前账号无权限时，自动回退到首个可访问页面，避免停留在越权页面；
4. 在用户管理页补充“工业系统总览”权限项，使权限配置与实际导航项保持一致。

### 价值

- 让文档中“16页面权限控制/菜单权限开关”与实际 UI 行为对齐；
- 降低不同角色误操作进入非授权页面的风险；
- 为后续继续细化按钮级权限、角色模板持久化提供基础。
