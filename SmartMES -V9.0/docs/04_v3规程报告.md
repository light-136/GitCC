## 二、设计评审

### 系统架构（四层）

```
SmartMES.UI          WPF表现层
  SmartMES.Services  服务层
    SmartMES.Modules 业务模块层
      SmartMES.Core  基础层（新增7个工业级子模块）
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| XAML绑定 | 纯TextBlock，禁用Run绑定 | Run在DataTemplate中DataContext继承失效→崩溃 |
| Brush绑定 | Freeze()+只绑Border.Background | ProgressBar.Foreground绑动态Brush→RewrapException |
| 线程模型 | DispatcherTimer+Task.Run+BeginInvoke | 彻底分离UI线程和后台线程 |
| 异常防护 | 三重全局捕获 | 防止任何异常导致闪退 |

### 新增Core模块结构

```
Core/Scheduler/     TaskSchedulerService  任务调度中心（系统心脏）
Core/StateMachine/  StateMachineEngine    工业状态机+守卫条件
Core/IO/            SimulatedIoDevice     16DI+16DO+4AI+4AO仿真
Core/Recipe/        RecipeService         多产品配方+JSON持久化
Core/Safety/        SafetyService         急停优先+条件互锁
Core/Traceability/  TraceService          SN+工序链+参数追溯
Core/Plugin/        PluginLoader          DLL动态加载+热插拔
```

## 六、全局异常防护机制

```csharp
// App.xaml.cs 三重保护
DispatcherUnhandledException += (_, e) =>
{
    ShowError("UI线程异常", e.Exception);
    e.Handled = true;  // 阻止闪退
};
TaskScheduler.UnobservedTaskException += (_, e) =>
{
    ShowError("后台任务异常", e.Exception);
    e.SetObserved();   // 阻止进程终止
};
AppDomain.CurrentDomain.UnhandledException += (_, e) =>
{
    // 兜底：至少弹窗提示，记录日志
};
```

## 八、后续建议

1. **接入真实硬件**: 将 `SimulatedIoDevice` 替换为 `ModbusIoDevice`/`OpcUaIoDevice`
2. **密码安全**: 生产环境使用 BCrypt 哈希存储密码
3. **数据库**: 生产环境使用 MySQL/SQL Server，配置连接字符串
4. **日志**: 接入 Serilog/NLog，支持结构化日志和远程推送
5. **配方持久化**: 调用 `RecipeService.SaveAsync()` 保存配方到 JSON
6. **插件扩展**: 在 `plugins/` 目录放置实现 `IPlugin` 接口的 DLL
7. **单元测试**: 为 AxisController/VisionEngine/StateMachineEngine 添加 xUnit 测试
