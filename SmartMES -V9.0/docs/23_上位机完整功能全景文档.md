# 上位机完整功能全景文档

> 文档定位：知识全景型 / 科普型 / 学习地图型文档  
> 目标：不局限于某一个项目，而是站在工业软件知识体系角度，尽可能完整地梳理一个“成熟上位机系统”可能具备的全部功能域、模块能力和工程能力，供学习、理解、分析与设计参考。  
> 阅读建议：把它当成“上位机世界地图”，而不是一份死记硬背的清单。

---

# 1. 先讲结论：上位机不是一个软件，而是一类系统

很多初学者第一次接触上位机时，会把它理解成：
- 一个桌面程序
- 一个设备控制界面
- 一个带按钮和图表的软件

这些理解都不算错，但都不完整。

更完整的定义是：

> **上位机是位于现场设备之上、管理系统之下，负责连接人、设备、流程、数据、质量与业务协同的现场执行与交互平台。**

它既是：
- 操作台
- 控制台
- 监视器
- 记录器
- 协调器
- 诊断终端
- 数据桥梁

所以如果你问“一个完整的上位机都应该有什么功能”，答案绝不只是几个页面，而是一整套能力体系。

# 3. 用户与界面功能域

这是用户最先接触到的部分。

## 3.1 典型功能

- 登录页
- 主界面
- 仪表盘
- 状态栏
- 当前用户信息
- 时间显示
- 系统标题与上下文显示
- 多语言切换（高级系统可选）
- 主题切换（可选）

## 3.2 这个功能域为什么存在

因为工业系统再复杂，也必须让人知道：
- 我现在看到的是什么
- 我当前能做什么
- 系统当前在干什么

## 3.3 学习重点

界面不是为了“好看”，而是为了：
- 清晰传达状态
- 降低误操作
- 缩短判断时间
- 提高可维护性

---

# 4. 导航与页面组织功能域

很多人低估导航，但实际上导航是大型上位机的人机骨架。

## 4.1 典型功能

- 左侧菜单导航
- 顶部页签导航
- 分组导航
- 面包屑导航（少见但可有）
- 页面标题同步
- 页面切换历史（可选）
- 默认首页
- 越权页面自动回退

## 4.2 为什么重要

因为工业项目页面通常很多：
- 监控页
- 参数页
- 日志页
- 报警页
- 控制页
- 配方页
- 数据页
- 权限页

没有清晰导航，用户很快会迷路。

## 4.3 成熟设计应考虑

- 页面分组是否符合业务场景
- 菜单是否按角色裁剪
- 当前页面是否清晰可见
- 页面切换是否稳定

这是上位机的神经网络。

## 5.1 常见能力

- 串口通信
- TCP 通信

- Modbus RTU
- Modbus TCP
- OPC UA
- MQTT
- HTTP REST
- WebSocket
- 厂商私有协议

## 5.2 典型子功能

- 建立连接
- 断开连接
- 自动重连
- 心跳
- 超时控制
- 报文发送接收
- 协议编解码
- 通信状态显示
- 通信日志
- 报文调试

## 5.3 为什么这是上位机第一基础能力

因为没有通信，上位机就无法连接现实设备与上层系统。

# 8. 状态机与运行模式功能域

这是工业软件和普通软件差异最大的关键之一。

## 8.1 状态机相关功能

- 状态定义
- 状态迁移
- 触发器
- 守卫条件
- 状态变化通知
- 标准设备状态模板

## 8.2 运行模式相关功能

- 手动模式
- 自动模式
- 调试模式
- 维护模式
- 教学模式（某些设备）
- 锁定模式

## 8.3 为什么一定要有

因为工业对象不能“随便执行任意动作”，它必须受状态和模式约束。

因为真实生产从来不是单动作，而是工艺步骤链。

## 14.1 典型功能

# 19. “完整功能”不等于“每个项目都全做”

这是非常重要的一点。

你看到这份清单可能会觉得：
“那是不是每个上位机都要这么全？”

不是。

## 19.1 不同上位机，侧重点不同

### 单机设备上位机
可能更重：
- 通信
- 控制
- 报警
- 参数
- 日志

### 视觉检测上位机
可能更重：
- 图像采集
- 检测算法
- 结果归档
- 追溯

### 产线上位机
可能更重：
- 多工位协同
- MES 接口
- 数据统计
- 权限与流程

### 工业演示/教学项目
可能更重：
- 模块覆盖面
- 可演示性
- 架构完整性

## 19.2 所以“完整功能地图”的意义是什么？

不是要求你一次做完所有，而是让你知道：

- 你现在做的是哪一类上位机
- 你已经覆盖了哪些功能域
- 你还缺哪些功能域
- 以后往哪扩展

# 20. 如果你想成为上位机高手，这份文档怎么用？

我建议你这样用。

## 第一步：把它当地图

先别急着记细节，只要先理解：
- 原来上位机有这么多功能域
- 原来它是一个系统，而不是几个页面

## 第二步：把每个项目往地图上贴

以后你看到任何上位机项目，都问自己：
- 它覆盖了哪些功能域？
- 它缺哪些？
- 它的重点在哪？

## 第三步：把自己想做的系统往地图上画

如果以后你自己设计系统，就按这张图思考：
- 我需要哪些域？
- 哪些是必须的？
- 哪些后续再做？

## 第四步：拿 SmartMES 做映射练习

你已经有 `SmartMES`，完全可以拿这张图反向对照：
- 哪些功能它已经做了
- 哪些是样例级实现
- 哪些还可以进阶

这样你学习会非常快。

# 22. 最后的学习建议

如果你以后想系统学习上位机，我建议你永远分三层去看：

## 第一层：功能层
它有什么功能？

## 第二层：模块层
这些功能背后有哪些模块？

## 第三层：系统层
这些模块如何协同，如何构成一个完整现场执行系统？

一旦你能同时看到这三层，你对上位机的理解就会真正上一个台阶。
