# SmartMES 注释补齐执行计划（全函数）

> 日期: 2026-04-02  
> 目标: 将业务代码中每个函数补齐可维护注释（不含 `obj/` 自动生成文件）

---

## 1. 范围定义

### 包含

- `SmartMES.Core/**/*.cs`
- `SmartMES.Modules/**/*.cs`
- `SmartMES.Services/**/*.cs`
- `SmartMES.UI/**/*.cs`（优先 ViewModel 和关键 Code-behind）
- `SmartMES.Tests/**/*.cs`（测试函数意图说明）

### 不包含

- `obj/**`、`bin/**` 自动生成文件
- 第三方包源码

---

## 2. 注释标准

每个函数至少包含：

1. **职责**：函数做什么；
2. **输入/输出**：关键参数含义与返回语义；
3. **副作用**：状态变更、事件触发、IO/日志影响；
4. **约束**：线程、时序、异常边界（如有）。

---

## 3. 分批执行

### 批次A（Core）

- Infrastructure / StateMachine / Scheduler / Safety / Recipe / Trace / IO / Plugin

### 批次B（Modules）

- MotionControl / Vision / VisionMotion / FileProcess / Database / MesComm / NativeInterop

### 批次C（Services）

- Logging / EventBus / Communication / Automation / Alarm / Device / Settings

### 批次D（UI）

- MainViewModel + 各 Module ViewModel + 关键 Code-behind

### 批次E（Tests）

- 测试函数注释（场景/预期/断言目标）

---

## 4. 验证策略

每个批次完成后执行：

1. `dotnet build SmartMES.sln`
2. `dotnet test SmartMES.sln -c Release`
3. 文档同步更新（设计文档 / 开发报告 / 测试报告）

---

## 5. 交付物

- 代码注释补齐后的源码
- 设计文档（`01_详细设计文档.md`）
- 项目开发报告（`02_项目规程报告.md` / `04_v3规程报告.md`）
- 快速启动指南（`03_快速启动指南.md`）
- 功能分级与函数职责说明（`09_工程功能分级与函数职责说明.md`）
