# SmartMES 单元测试使用说明

> 日期: 2026-04-02  
> 适用项目: `SmartMES.Tests`

---

## 1. 前置条件

1. 已安装 .NET SDK 9+
2. 当前目录位于项目根目录：`SmartMES/`

---

## 2. 常用命令

### 2.1 运行全部单元测试

```powershell
dotnet test SmartMES.sln -c Release
```

### 2.2 仅运行测试工程

```powershell
dotnet test SmartMES.Tests/SmartMES.Tests.csproj -c Release
```

### 2.3 按类过滤运行

```powershell
dotnet test SmartMES.Tests/SmartMES.Tests.csproj -c Release --filter "FullyQualifiedName~StateMachineEngineTests"
```

### 2.4 采集覆盖率（Cobertura）

```powershell
dotnet test SmartMES.Tests/SmartMES.Tests.csproj -c Release --collect:"XPlat Code Coverage"
```

---

## 3. 新增测试用例规范

1. 测试文件命名：`<被测类>Tests.cs`
2. 测试方法命名：`行为_条件_期望结果`
3. 单测原则：
   - 单一断言目标（可有多个Assert，但围绕一个行为）
   - 明确 Arrange / Act / Assert
   - 避免依赖 UI 线程与外部设备

---

## 4. 故障排查

### 4.1 测试卡住或超时

- 检查是否存在未结束的进程；
- 检查异步等待是否设置超时；
- 先按类过滤定位问题用例。

### 4.2 构建通过但测试失败

- 优先查看失败断言与堆栈；
- 判断是预期变更还是代码缺陷；
- 若是行为变更，应同步更新测试与设计文档。

---

## 5. 与CI集成建议

在后续CI中增加以下步骤：

1. `dotnet restore`
2. `dotnet build -c Release`
3. `dotnet test -c Release`
4. （可选）上传覆盖率报告
