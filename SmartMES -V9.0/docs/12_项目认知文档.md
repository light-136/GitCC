# SmartMES 项目认知文档

> 目的：帮助阅读者快速理解 `SmartMES` 是什么项目、解决什么问题、采用了什么技术和架构、主要模块有哪些、当前项目处于什么成熟度。

## 1. 项目概述

`SmartMES` 是一个基于 `WPF + .NET 9` 开发的**智能制造上位机 / MES 综合演示平台**。

它不是单一功能软件，而是围绕制造现场常见需求，集成了以下能力：

- 设备监控
- 通信管理
- 报警与日志
- 自动化流程
- MES 通信
- 数据库接入与统计
- 文件处理与导出
- 运动控制
- 视觉检测
- 配方管理
- 安全互锁
- 追溯系统
- 插件化扩展
- C++ 原生互操作

从项目目录、解决方案、文档和核心代码判断，这个项目既是一个**工业上位机样板工程**，也是一个**可扩展的智能制造演示平台**。

---

## 2. 这个项目是什么类型

从整体设计来看，`SmartMES` 更偏向以下定位：

1. **工业智能制造上位机系统**
2. **MES/设备控制综合演示平台**
3. **工业软件分层架构样板工程**
4. **可用于二次开发的基础平台**

它不是传统 Web 型 MES，也不是纯数据库后台系统，而是一个运行在 Windows 桌面环境中的工业控制与业务协同平台。

---

## 3. 项目面向的业务场景

这个项目主要适用于以下场景：

- 产线设备状态监控

- 工艺参数 / 配方管理
- 数据统计、导出与报表展示
- 工业模块能力集成展示

因此，它既适合：

- 课程/课题演示
- 技术架构展示

- 后续真实项目二次开发

## 5. 解决方案结构认知

解决方案 `SmartMES.sln` 中包含以下项目：

- `SmartMES.Core`
- `SmartMES.Services`
- `SmartMES.Modules`
- `SmartMES.UI`
- `SmartMES.Tests`
- `SmartMES.Native`

这是一个典型的**分层多项目解决方案**。

---

## 6. 分层架构说明

### 6.1 `SmartMES.UI`

**职责：表现层**

负责：

- WPF 页面展示
- 用户交互
- 左侧菜单导航
- View / ViewModel 绑定
- 权限控制下的页面显隐
- 模块页面组织

这是系统面向用户的界面层。

---

### 6.2 `SmartMES.Services`

**职责：系统服务层**

负责：

- 报警服务
- 日志服务
- 事件总线
- 通信服务
- 自动化流程引擎
- 设备服务
- 参数设置服务

这是系统级公共服务聚合层。

---

### 6.3 `SmartMES.Modules`

**职责：业务模块层**

负责：

- 数据库模块
- 文件处理模块
- MES 通信模块
- 运动控制模块
- 原生互操作模块
- 视觉检测模块
- 视觉+运动协同模块

这是项目中具体业务能力的主要实现层。

---

### 6.4 `SmartMES.Core`

**职责：核心基础层**

负责：

- MVVM 基础设施
- 接口定义
- 状态机引擎
- 调度中心
- IO 抽象
- 配方系统
- 安全互锁
- 追溯系统
- 插件化基础设施

这是整个系统的底层能力层。

---

### 6.5 `SmartMES.Tests`

**职责：测试层**

负责对核心功能进行单元测试，包括：

- 状态机
- 调度器
- 配方服务
- 运动控制
- 权限控制
- 安全服务
- 追溯服务

---

### 6.6 `SmartMES.Native`

**职责：原生扩展层**

这是一个 C++ 动态库工程，配合 `P/Invoke` 被 `C#` 调用，用于展示：

- 数学计算
- 矩阵运算
- 数组处理
- 字符串处理
- 原生模块与托管模块协同

## 7. 主要功能模块认知

### 7.1 监控与基础功能

根据 README 和 UI 导航设计，系统包含以下基础监控能力：

- 实时仪表盘
- 设备管理
- 通信监控
- 报警管理
- 系统日志
- 自动化流程

这些模块构成典型工业上位机的日常操作区域。

---

### 7.2 MES 通信模块

目录：`SmartMES.Modules/MesComm`

包含：

- `HttpMesClient`
- `MqttMesClient`
- `OpcUaMesClient`

说明项目考虑了多种工业/系统通信方式。

当前从代码实现判断，该模块具有明显的**模拟通信演示**属性，例如：

- 模拟连接 MES
- 模拟获取工单列表
- 模拟生产结果上报
- 模拟后台轮询推送

结论：这是一个**对接能力骨架 + 示例实现**，适合作为后续真实接口开发基础。

---

### 7.3 数据库模块

目录：`SmartMES.Modules/Database`

支持：

- `SQLite`
- `MySQL`
- `SQL Server`

具备以下特征：

- 基于 `EF Core` 的 `DbContext`
- 生产记录实体 `ProductionRecord`
- 设备日志实体 `DeviceLog`
- 多数据库可切换
- 适合演示自动建表、统计分析、数据导出

说明该模块主要承担本地或企业数据库的数据管理与分析能力。

---

### 7.4 文件处理模块

目录：`SmartMES.Modules/FileProcess`

从项目说明判断，支持多格式读写与批量导出，主要用于：

- 结果导出
- 报表导出
- Excel 处理
- JSON/CSV 文件生成

该模块提升了系统的数据交换与报表输出能力。

---

### 7.5 运动控制模块

目录：`SmartMES.Modules/MotionControl`

包含：

- `AxisController`
- `MultiAxisController`
- `TenAxisController`

从 `AxisController` 看，支持：

- 回零
- 绝对定位移动
- 暂停 / 恢复
- 停止
- 错误置位
- 复位

说明该模块是一个**多轴运动控制逻辑模拟与抽象层**，适合用于：

- 控制流程演示
- 轴状态管理
- UI 实时联动展示
- G 代码执行示例

当前更像“工业控制逻辑样板”，而非直接对接真实运动控制卡驱动的底层实现。

---

### 7.6 10 轴模块

UI 中存在：`10轴版面切换`

文档说明其具备：

- G 代码编辑
- 程序执行
- 10 轴状态展示
- 5×2 / 2×5 布局切换
- 急停 / 复位 / 回零操作

这是一个很强的演示模块，适合展示复杂设备控制界面的组织能力。

---

### 7.7 视觉检测模块

目录：`SmartMES.Modules/Vision`

从 `VisionEngine` 判断，其当前是**纯 C# 视觉示例模块**，支持：

- 生成模拟工件图像
- 灰度处理
- Sobel 边缘检测
- 基于阈值的缺陷检测
- 标注缺陷框
- 输出 OK / NG 判定

这说明它更偏：

- 视觉流程演示
- 缺陷检测概念验证
- 界面展示与教学示例

而非完整工业视觉产线方案。

---

### 7.8 视觉 + 运动协同模块

目录：`SmartMES.Modules/VisionMotion`

该模块体现的是多工位、多任务场景下的协同调度思想，用于展示：

- 视觉检测与运动执行联动
- 多工位调度
- 并行任务组织

这类模块很符合智能制造现场的综合控制需求。

---

### 7.9 原生互操作模块

目录：`SmartMES.Modules/NativeInterop` 与 `SmartMES.Native`

通过 `P/Invoke` 调用 C++ DLL，提供：

- 平均值 / 标准差 / 中位数
- 数组排序
- 矩阵乘法 / 转置 / 加法
- 移动平均
- 欧式距离
- 字符串处理

说明项目具备：

- 托管代码与原生代码协同设计能力
- 将计算型逻辑下沉到 C++ 的扩展能力
- 混合技术栈工业桌面应用的实现基础

## 8. 工业系统特征分析

这个项目有明显的工业软件特征，而不是普通管理系统。

### 8.1 状态机引擎

`StateMachineEngine` 支持：

- `Idle`
- `Running`
- `Paused`
- `Alarm`
- `Resetting`
- `Error`

并支持：

- 触发器
- 状态迁移
- 守卫条件
- 状态切换事件

这说明项目在核心控制逻辑上具有标准工业设备状态建模能力。

---

### 8.2 调度系统

`TaskSchedulerService` 支持：

- 周期任务
- 延迟任务
- 一次任务
- 优先级任务
- 任务状态记录
- 完成/失败事件

这说明系统具备工业后台任务调度能力，适合处理：

- 轮询
- 定时采集
- 自动动作触发
- 高优先级紧急任务

---

### 8.3 追溯系统

`TraceService` 支持：

- 产品 SN 追溯
- 工序记录
- 工位/设备记录
- 参数与结果记录
- 时间范围查询
- 最终 OK/NG 判断

这是典型制造执行系统中非常关键的一块能力。

---

### 8.4 配方系统

`RecipeService` 存在独立实现和单元测试，说明项目具备工艺参数管理思路，符合制造场景中的“产品配方 / 工艺参数集”概念。

---

### 8.5 安全互锁

存在 `SafetyService`，说明系统已经考虑安全控制逻辑，是工业控制领域的重要设计点。

---

### 8.6 权限系统

项目内置多角色账号：

- `admin`
- `operator`
- `viewer`

并且 UI 层对页面访问做了权限判断，说明系统有基本的角色权限管理能力。

## 11. 项目成熟度判断

### 11.1 优势

该项目的成熟点主要体现在：

- 结构清晰
- 模块丰富
- 分层明确
- 文档较多
- 测试覆盖较完整
- 技术栈组合合理
- 工业特征明显

### 11.2 当前阶段判断

从代码实现来看，项目目前更偏向：

- **高质量演示平台**
- **工业软件架构样板**
- **原型系统 / 二次开发底座**

而不是已经完成全部真实工业设备接入和投产部署的最终生产系统。

原因包括：

- 多个模块带有模拟实现特征
- 视觉模块偏示例性质
- MES 通信存在演示化实现
- 运动控制更偏逻辑模拟而非真实硬件驱动封装

因此最准确的定位是：

> 一个面向智能制造场景的综合型上位机平台样板，具备工业系统核心能力抽象，适合演示、教学、原型验证和后续工程扩展。

## 13. 当前存在的注意点

在阅读中也发现一些需要注意的地方：

1. **部分中文注释存在编码异常**
   - 个别文件中出现中文乱码，推测是文件编码历史不统一造成。
   - 不严重影响理解，但会影响维护体验。

2. **部分模块偏模拟实现**
   - 如果要真正投产，还需要补足真实设备接入、接口联调、现场稳定性验证等工作。

3. **平台绑定 Windows**
   - 由于使用 `WPF`，这是一个典型 Windows 桌面工业应用，不是跨平台客户端。
